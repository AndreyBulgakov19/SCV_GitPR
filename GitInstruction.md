# Работа с Git

## 1. Проверка наличия установленного Git
В терминале выполнить команду `git version`.
Если Git установлен появится сообщение с информацией о версии программы, иначе будет сообщение об ошибке.

## 2. Установ ка Git
Загружаем последнюю версию Git с [сайта](https://git-scm.com/downloads).
Устанавливаем с настройками по умолчанию.

## 3. Настройка Git
При первом использовании Git необходимо пердставиться.
Для этого нужно ввести в терминали 2 команды:
```
git config --global user.name "Ваше имя английскими буквами"

git cinfig --global user.email "ваша почта@example.ru"
```

## 4. Создание нового репозитория

Что бы создать новый репозиторий, необходимо открыть папку нашего проекта через проводник Visual Studio Code и в терминале выполнить команду `git init`. Данная команда включит приложение Git в конкретной папке и создаст скрытую директорию .git где будет храниться история репозитрия и настройки.

## 5. Определение состояние репозитория

Для определения состояния репозитория используется команда `git status`, это одна из важнейших команд, которая показывает информацию о текущем состоянии репозитория: актуальна ли информация в нём, нет ои чего-то нового, что поменялось и т.д.
Запуск команды `git status` в новом репозитории должен выдать следующий ответ:
```
$ git status
On branch master
Initial commit
Untracked files:
(use "git add ..." to include in what will be committed)
(название файла)
```
Сообщение говорит о том что, что файл созданный в папке проекта неотслеживаемый и система не знает нужно следить за изменениями в файле или его можно просто игнорировать.

## 6. Подготовка и сохранение файлов

В git есть концепция области подготовленных файлов-коммит. Можно представить ее как холст, на который наносят изменения, которые нужны в коммите. Сперва он пустой, но затем мы добавляем на него файлы (или части файлов, или даже одиночные строчки) командой `git add` и, наконец, коммитим все нужное в репозиторий (создаем слепок нужного нам состояния) командой `git commit`. Эти команды можно объединить в одну с помощью следующей команды `git commit -a -m`.

Написание исполенения команд выглядит следующим образом:
```
git add "навзвание файла"

git commit -m "название коммита или комментарий к данному сохранению"

git commit -a -m "название коммита или комментарий к данному сохранению"

```

## 7. Просмотр сохранений и изменений

Для просмотра всех выполненных фиксаций можно воспользоваться историей коммитов. Она содержит сведения о каждом проведенном коммите проекта. Запросить ее можно при помощи команды `git log`. В ней содержится вся информация о каждом отдельном коммите, с указанием его хэша, автора, списка изменений и даты, когда они были сделаны.

## 8. Переключение между сохранениями или коммитами

Git позволяет вернуть выбранный файл к состоянию на момент определенного коммита. Это делается командой `git checkout`.
Для использования данной команды необходимо сначала получить список всех коммитов с помощью команды `git log` и далее использовать команду `checkout` следующим образом:
```
git checkout "номер или код коммита"
```

## 9. Игнорирование файлов
Для того, чтобы исключить из отслеживания в репозитории опредленные файлы или паки необходимо создать там файл ***.gitignore*** и записать в него их названия или шаблоны, соответсвующие таким файлам или папкам.

## 10. Создание веток в Git
По умолчанию имя основной ветки в Git - **main**.
Создать ветку можно командой:
```
git branch <имя новой ветки>
```
Список веток в репозитории можно посомтреть с помощью команды `git branch`. Текущая ветка будет отмечена звездочкой: **\* main**.

Для переключение между ветками можно использовать следующие команды:
```
git checkout <имя ветки>

git switch <имя ветки>
```

## 11. Слияние веток и разрешение конфликтов при слиянии
Когда вы работаете в одной из веток информация и изменения происходят только в текущей ветке, для слияния веток необходимо использовать команду `git merge` следующим образом: 
* сохраните текущие изменения в вашей ветке c помощью команды `commit` и переключитесь в ветку с которой вы хотите произвести слияние с помощью команды `checkout` или `switch`;
* теперь с помощью команды `merge` можно произвести слияние необходимых вам веток, введите команду в терминале следующим образом:
```
git merge <имя ветки>
```
После чего произойдет полное слияние двух ваших веток и информация в ветках станет идентичная.

Иногда при слиянии могут возникат конфликты, когда информация в двух ветках имеет разный вид и содержание, тогда Git  предложит вам выбрать необходимый итоговый вариант, если вы используете VSC то данное действие будет выглядит следующим образом:
![error](prinscreen.png)

Здесь вы должны выбрать принять текущее изменение или принять входящее изменение, после чего ветки сольются.

После слияния веток необходимо обязательно произвести коммит слияния с помощью команды `git commit`.

## 12. Удаление веток

Бывают ситуации, когда после слива каких-то изменений из рабочей ветки в исходную версию проекта, ее, по правилам хорошего тона, необходимо удалить, чтобы она более не мешалась в вашем коде. Но как это сделать?
Для локально расположенных веток существует команда:

```
git branch -d <имя ветки>
```
где флажок -d являющийся опцией команды git branch - это сокращенная версия ключевого слова --delete, предназначенного для удаления ветки. Если в ветке есть несохраненные изменения Git не удалит ветку и сообщит вам о данной ситуации и предложит вариант принудительного удаления ветки без сохранения изменений, данной действие можно произвести заменой в команде -d на -D.

Однако тут есть нюанс: удалить текущую ветку, в которую вы, в данный момент просматриваете - нельзя. Если же вы все-таки попытаетесь это сделать, система отругает вас и выдаст ошибку. Так что при удалении ветвей, обязательно переключитесь на другой ветку. 

## 13. Подключение к удаленному репозиторию

Чтобы загрузить что-нибудь в удаленный репозиторий, сначала нужно к нему подключиться. Зарегистрируйте аккаунт на портале [GitHub](https://github.com) и после регистрации на портале создайте свой удаленный репозитрий с помощью кнопки + и далее `New repository`.

Для того чтобы связать наш локальный репозиторий с репозиторием на GitHub, выполним следующую команду в терминале. Обратите внимание, что нужно обязательно изменить URL репозитория на свой:
```
git remote add origin https://github.com/tutorialzine/awesome-project.git
```

## 14. Отправка изменений на сервер

Сейчас самое время переслать наш локальный коммит на сервер. Этот процесс происходит каждый раз, когда мы хотим обновить данные в удаленном репозитории.
Команда, предназначенная для этого - `git push`. 

Ответ на данную команду будет выглядеть так:
```
git push                           
Enumerating objects: 8, done.
Counting objects: 100% (8/8), done.
Delta compression using up to 8 threads
Compressing objects: 100% (4/4), done.
Writing objects: 100% (6/6), 624 bytes | 624.00 KiB/s, done.
Total 6 (delta 2), reused 0 (delta 0), pack-reused 0
remote: Resolving deltas: 100% (2/2), completed with 1 local object.
To https://github.com/mrgpnv/GitHub1.git
   af9dffb..dbd7374  main -> main
```

После исполнения команды вы сможете увидеть данный коммит уже в своем репозитории на портале GitHub.

Однако пушить надо осторожно, ведь рассматриваемая команда перезаписывает безвозвратно все изменения. В большинстве случаев, ее используют, чтобы опубликовать выгружаемые локальные изменения в центральный репозиторий. А еще ее применяют для того, чтобы поделиться, внесенными в локальный репозиторий, нововведениями, с коллегами или другими удаленными участниками разработки проекта. В зависимости от сервиса, который вы используете, вам может потребоваться аутентифицироваться, чтобы изменения отправились. Если все сделано правильно, то когда вы посмотрите в удаленный репозиторий при помощи браузера, вы увидите файл свой файл.

## 15. Запрос изменений с удаленного репозитория

Если вы сделали изменения в вашем удаленном репозитории, другие пользователи могут скачать изменения при помощи команды `git pull`.

Ответ на данную команду будет выглядеть так:
```
git pull
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Compressing objects: 100% (2/2), done.
Total 3 (delta 1), reused 0 (delta 0), pack-reused 0
Unpacking objects: 100% (3/3), 709 bytes | 141.00 KiB/s, done.
From https://github.com/mrgpnv/GitHub1
   ccc6fa1..af9dffb  main       -> origin/main
```

В случае если информация в репозиториях отличается возникнит кофнликт, решение конфликтов описано в п. 11.

## 16. Как удалить локальный репозиторий

Если вам не понравился один из ваших локальных Git-репозиториев и вы хотите стереть его со своей машины. Для этого вам всего лишь надо удалить скрытую папку «.git» в корневом каталоге репозитория. Сделать это можно 3 способами:

1. Проще всего вручную удалить эту папку «.git» в корневом каталоге «Git Local Warehouse».
2. Также удалить, не устраивающий вас, репозиторий можно на github. Открываете нужный вам объект и переходите в пункт меню Настройки. Там, прокрутив ползунок вниз, вы попадете в зону опасности, где один из пунктов будет называться «удаление этого хранилища».
3. Последний метод удаления локального хранилища через командную строку, для этого в терминале необходимо ввести следующую команду:

```
cd repository-path/
rm -r .git
```
