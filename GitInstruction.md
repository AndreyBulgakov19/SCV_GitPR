# Работа с Git

## 1. Проверка наличия установленного Git
В терминале выполнить команду 
```
git --version
```
Если Git установлен появится информация о версии программы, иначе появится сообщение об ошибки.

## 2. Установка Git 
Загружаем последнюю версию программы с [сайта](https://git-scm.com/downloads).
Устанавливаем с настройками по умолчанию. 

## 3. Настройка Git 
Я работаю в программе **VSC**, вы можете открыть терминал где вам вам удобно.



#### Теперь, если вы первый раз установили Git, в терминале нужно прописать 2 команды:

1. Сказать git как вас зовут, через команду 
```
git config --global user.name «Ваше имя английскими буквами»
```
2. И предоставить свой email: 
```
git config --global user.email вашаПочта@example.com
```
*Сделать это нужно всего лишь один раз, и git вас запомнит.* 

## 4. Инициализация репозитория
#### Для начала работы с **Git** нужно создать папку, где будет храниться ваш репозиторий. 

Как пример, я сделал это на рабочем столе, и моя папка называется **"Git"**. 

Если вы как и я работаете через **"VSC"**, в проводнике нужно открыть только что созданную папку "Git". 

В этой папке нам нужно оздать файл, придумайте название этого файла которое относилось бы к нашей теме, и ***Важно сделать расшерение файла  .md***

##### Приступим к созданию репозитория

Прописываем в терминале следующую команду: 
```
git init
```

## 5. Запись и измненение репозитория

Давайте напишем в этом файле небольшую информацию о себе, например: 
1. Как вас зовут
2. Сколько вам лет 

Теперь нам нужно добавить файл к следующему коммиту, через команду

#### Если нужно добавить все файлы:

```
git add . - добавление всех файлов
``` 
#### Если нужно добавить конкретный файл: 
```
git add "название файла"
```
Далее нам нужно создать коммит и в скобках указать комментарий, например "какие изменения были сделаны", через следующую команду: 
``` 
git commit -m "комментарий"
```
Так же, есть команда, которая включается в себя сразу добавления файлов и создание коммита, но нужно знать то что *она сделает это со всеми файлами с которыми вы работали*.
```
git commit -am "комментарий"
```

## 6. Просмотр изменения коммитов
#### Далее список команд которые важно знать: 


Через следующую команду можно посмтореть на историю всех коммитов с их хеш-кодами
```
git log 
```
есть еще одна команда, которая может показать история коммитов в одну линию
```
git log --oneline
```
Хочу сказать об одной команде которая дает информацию от git о его текущем состоянии
```
git status
```
Если вы хотите увидеть разницу между теующим файлом и закомиченным файлом, вам нужна команда 
```
git diff
```

## 7. Перемещение между сохранениями/коммитами 
Уверен в работе вам пригодится команда которая переключает от одного коммита к другому.
```
git checkout 
```
Если вы захотите вернуться на main ветку, к актуальному состоянию и продолжить работу, то введите следующую команду: 
```
git checkout main
```
## 8. Игнорирование файлов 
Для того, чтобы исключить из отслеживания в репозиториии опредленные файлы или папки необходимо создать файл ***.gitignore *** и записать в него их названия или шаблоны, соответствующие таким файлам или папкам. 

## 9. Создание веток в Git
По умолчанию имя основной ветки в Git - **main** 
Создать ветку можно командой: 
```bash 
git branch <имя новой ветки>
```
Список веток в репозиториии можно посмотреть с помощью команды: 
```bash
git branch 
```
## 10. Слияние веток и разрешение конфликтов 
Для того чтобы слить ветку в которой вы тестировали новый функционал:
1. Нужно перейти на **main** ветку
2. Прописать следущую команду 
```
git merge name_branch
```
3. Сделать commit 

При слиянии у вас может возникнуть *конфликт*, если в однинаковых блоках в ветке **main** и в ветке **name_brunch** у вас находится информация

В зависимости от программы в который вы работаете, вы можете оставить информацию из *ветки 1* или из *ветки 2* или оставить информацию из *обоих веток*

## 11. Удаление веток 
Если вы слили ветки и ветка которая была черновиком вам не нужна, вы можете ее удалить
Для это нужно прописать в консоли следующую команду: 
```
git branch -d name_brunch 
```
Чтобы удалить не слитую ветку, вам потребуется следующая команда:
```
git branch -D name_branch
```
Важно знать: __***нельзя удалить ветку находясь в ней***__

## 12. Работа с удаленным репозиторием
a) Создать аккаунтна на GitHub

b) Создать локальный репозиторий 

c) Создать удаленный репозиторий

1. На сайте GitHub зайти в "Your repositories"
2. Нажать на копку "New"
3. Написать название вашего репоитория в онке "Repositiry Name"
4. Выбрать будет он "Public" or "Private"
5. Выбрать будете ли вы создавать его уже с файлом "README" и т.д.
6. Нажать на кнопку "Create repository"

d) Связать локальный репозиторий с удаленным

В терминале прописываем следующие команды: 
```
git remote add origin REMOTE-URL
```
```
git branc -M main
```
```
git push -u origin main
```
*Если вы первый раз проделываете эту опреацию, вам нужно авторизоваться в VSC через аккаунт GitHub, сама программа вам в этом поможет, просто следуйте инструкции. Это нужно сделать всего лишь один раз, дальше VSC вас запомнит!

### Несолько полезных команд: 
##### Отправить локальную версию репозитория на удаленный 

После такого, как вы выполнили определнную работу, вам нужно отправить локальный репозиторий на удаленный, это делается через следующую команду. 
```
git push
```

###### Получить изменения из удаленного репозитория и выполнить слияние с локальной версией

Если же вы проделали определнную работу на сайте GitHub, добавили изменения, сделали коммиты. Вам нужно синхронизировать то что у вас есть на удаленном репозиторие с локальным, это делается через следующую команду: 

```swift
git pull
```

Проверка удаленных репозиториев 
```
git remote
```
Подробное название удаленных репозиториев 
```
git remote -v
```

### Fork, Pull Request
Допустим вы нашли проект на GitHub, изучили код и увидели что можете быть полезным, предложить какую-то идею, сделать фитчу или исправить определнную ошибку.

Как сделать? Како отправить свою работу в их проект? 

На сайте GitHub находите репозиторий например [octocat/Spoon-Knife](https://github.com/octocat/Spoon-Knife)

В верхней-правой части, нажимаете на кнопку "Fork"
Далее нажимаете на кнопку "Create repository"

Откроется Репозиторий проекта но уже на вашем аккаунте, там же вы и увидите что это репозиторий от аккаунта "octocat"

Нажимаете на кнопку "Code", копируете ссылку этого репозитория и переходите в терминал 

###### Теперь нам нужно клонировать этот проект на наш локальный репозиторий!

В терминале прописываете команду 
```
git clone https://github.com/DanielIganshin/Spoon-Knife.git
```
!Обязательно создаете новую ветку и работаете в ней

Делаете изменения, создаете файлы, исправляете баги, делаете фитчи. Дальше делаете коммит с сообщением, отправляете репозиторий через команду `git push`

###### Вы сделали изменения в этом проекте, теперь вам нужно эти изменения отправить в сам проект, далее описано как это сделать:

Вернитесь на github, обновите страницу и у вас появиться кнопка "Compare and pull request"

С помощью этой кнопки вы сможете отправить свою работу в их проект, теперь команда проверит то что вы сделали и если она посчитает нужным  -> внесет в проект "Spoon-knife"!

*Примичание 

* Если вы не нашли эту кнопку, вам нужно перейти во вкладку "pull request" 
* В правом верхнем углу выбрать "New pull request"
* Далее вы увидите зеленую кнопку "Create pull request"
* Указываете название коммита, так же можете написать свой комментарий
* И еще раз нажимаете "Create pull request"
