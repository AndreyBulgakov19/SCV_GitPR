# Job for git

## 1. Проверка установленного git
В терминале выполнить команду `--git version`
Если git установлен появится сообщение с информацией о версии программы, иначе будет сообщение об ошибке.

## 2. Установка GIT
Загружаем последнюю версию git с [сайта](https://git-scm.com/downloads)
Устанавливаем с настрйоками по умолчанию 

## 3. Настройка Git
При первом использовании Git необходимо представиться.
Для этого нужно ввести в терминале 2 команды:
```
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```

## 4. Инициализация репозитория
Чтобы начать слежение за текущим проектом, первое что нужно сделать это ввести команду в терминал `git init`. Если все ранее было выполнено правильно - В результате в существующей папке появится еще одна папка с именем .git и всеминужными вам файлами репозитория — это будет основа вашего Git-репозитория

## 5. Cоздание коммитов

Чтобы начать управление версиями существующих файлов укажите файлы, за которыми должна следить система, и выполните
первую фиксацию изменений. Для этого потребуется несколько команд `git add "имя файла"` , добавляющих файлы, за которыми вы хотите следить, а затем команда `git commit -m ""`
В дальнейшем можно использовать составную команду  git add. + git commit:
```
git commit -am ""
```
Для изменения комментария последнего коммита используется команда:
```
git commit --amend -m ""
```
Для внесения изменений последнего коммита с комментарием ипользуется:
```
git commit --amend -am ""
```
## 6. Просмотр состояния репозитория 
 
Основным инструментом определения состояния файлов является команда
`git status` после введения этой команды вы увидете были ли измениния в файле или нет.
Для просмотра сокращенной версии отчета можно использовать команды:
`git status -s` или `git status --shorts`

Для просмотра состояния индексированных и неиндексированных изменений используется команда `git diff`

Для просмотра разницы между индексом и последним коммитом используется команда:
```
git diff --staged
```
## 7. Просмотр историй коммитов

После того, как вы создали несколько коммитов , вероятно вам понадобится возможность посмотреть что было сделано — историю коммитов. Для этого стоит вписать в терминал команду `git log`. **git log** перечисляет коммиты, сделанные в репозитории в обратном хронологическом порядке — последние коммиты находятся вверху.
Для просмотра в укороченном формате:
```
git log --oneline
```
Для просмотра историй с коммитами и слиями веток([**Глава 11**](#11-слияние-веток-и-разрешение-конфликтов)) в виде древа используется команда:
```
git log --graph 
``` 
Опция `oneline` являются особенно полезными с опцией --graph команды log
вы сможете увидеть небольшой граф, который показывает текущую ветку и историю слияний:

## 8. Перемещение между сохранениями(коммитами)

Для перемещениями между сохранениями(коммитами) используется команда `git checkout <код коммита>`

## 9. Игнорирование файлов 

Для того, чтобы исключить из отслеживания в репозитории определенные файлы или папка, необхадимо создать там файл ***.gitignore*** и записать в него их названия или шаблоны, соответствующие таким файлам или папкам.

## 10. Создание веток в Git
По умолчанию имя основной ветки в Git - **master**
Чтобы создать новую ветку  нужно вписать в терминал команду:
```
git branch <имя новой ветки>
```
или для создания и автоматического перехода на эту ветку можно использовать команду:
```
git checkout -b <имя новой ветки>
```
Чтобы просмотреть список веток в репозитории нужно вписать команду  `git branch`. 
Текущая ветка будет отмечена : **\*master**

## 11. Слияние веток и разрешение конфликтов

Для слияния веток нужно:переключиться на ветку, в которую вы хотите включить изменения, и выполнить команду:
```
git merge <Имя ветки>
```
Иногда процесс не проходит гладко. Если вы изменили одну и ту же часть одного и того же файла по-разному в двух объединяемых ветках, Git не сможет их чисто объединить.Если вы работаете в VSCode он предложит вам решения конфликта который вы должны выбрать сами.

После разрешения кофликта слияний необходимость сделать коммит.

## 12. Удаление веток
Для удаление веток в репозитории используется команда:
```
git branch -d <Имя ветки>
```
-d сокращенный вариант записи --delete. Как и следует из названия удаляет ветки

Если в ветке присутсвуют не слитые измения -d не позволит ее удалить 
Для принудительного удаления ветки из репозитории используется команда :
```
git branch -D <Имя ветки>
```
# Работа с удаленными репозиториями

## 1. GitHub

GitHub — это облачная платформа для хостинга IT-проектов и совместной разработки, основанная на основе популярой системе контроля версий Git, а также полноценная социальная сеть для разработчиков.

* Для начала работы необходимо создать учетную запись на [GitHub](https://github.com/).
* Далее мы можем создавать новый репозиторий или работать уже над любым созданным обьектом с открытым для нас репозиторием.

## 2. Создание нового репозитория

Для создания репозитория нужно нажать на + в правом верхнем углу сайта, выбрать пункт New Repository, заполнить название и описание, проставить нужные галочки и щёлкнуть на Create Repository. Откроется страница создания нового репозитория. Необходимо добавить наименование нового репозитория, и если одноименных репозиториев на этом аккаунте нет, то имя будет доступно. Также, опционально можно добавить описание , тип доступа (публичный или приватный), добавить файл readme(описание проекта) и добавить gitignore свой или по шаблону.

## 3. Связывание локального и удаленного репозитория:

* Необходимо связать локальный репозиторий с удаленным для этого используется команда:
```
git remote add origin <ссылка на удаленный репозиторий>
``` 
Вводим эту комманду в локальном репозитории, тем самым связываем локальный и удаленный репозиторий. Проверяем связь при помощи команды `git remote` в локальном репозитории.
Для получения более подробной информации добавляем функцию -v так мы дополнительно получим ссылки(пути) для получения и отправки данных.
* Если необходимо переименуем master в main с помощью команды 
```
git branch -M main
```
Отправляем изменения в удаленный репозиторий при помощи команды:
 ```
 git push -u origin main
 ```
 При первой отправке git попросит авторизоваться.

## 3. Редактирование удаленного репозитория и получение изменений из удаленного репозитория

* Для редактирования на самом GitHub необходимо нажанать на иконку карандаша. После завершения изменений нажанать на _`commit changes`_

* Для получения изменений из удаленного репозитория, переходим в локальный репозиторий и применяем команду `git pull`
* В случае возникновения конфликта, действуем так же как и при конфликте слияния локальных веток.[**Глава11**](#11-слияние-веток-и-разрешение-конфликтов)

* В ситуации когда изменения вносились в удаленный репозиторий, необходимо сначала подгрузить изменения из удаленного в локальный, решить конфликты, а потом отправлять готовые данные обратно в удаленный репозиторий.

## 4. Работа с чужими репозитоиями.
* Вводим в поисковой строке название нужного репозитория или поисковое слово.
* Можно отсортировать результаты поиска по содержанию(код(выбрать язык программирования), репозитории, обсуждения и т.д.)
* После нахождения нужного репозитория, открываем его, нажимаем кнопку code и выбираем вкладку local и выбираем https и копируем ссылку.
* Переходим в редактор кода и используем команду `git clone` в пустой папке.
* Папка с репозиторием появится на компьютере, переходим в папку: выделяем папку-ПКМ-открыть во встроенном терминале.
* Перед началом работы с чужим репозиторием нужно создать его копию в своем аккаунте при помощи кнопки fork
