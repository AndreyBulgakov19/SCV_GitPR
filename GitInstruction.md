# Работа с Git

## 1. Проверка наличия установленного Git

В терминале выполнить команду `Git --version`. Если Git установлен, появится сообщение с игформацией о версии программы, иначе будет сообщение об ошибке.

## 2. Установка Git
Загружаем последнюю версию с [сайта](https://git-scm.com/). Устанавливаем с настройками по умолчанию.

## 3. Настройка Git

При первом использовании Git необходимо представиться. Для этого нужно ввести в терминале две команды:
```Bash
git config --global user.name <Ваше имя английскими буквами>
git config --global user.email ваша почта почта@email.com
```
## 4. Инициализация репозитория

**Наша работа в Git начинается с выбора нашей рабочей папки, далее мы выполняем нашу первую  команду _git init_, тем самым мы инициализируем наш локальный репозиторий, для дальнейшей работы в нашем репозитории**. 

## 5. Просмотр ожидающих файлов в репозиторий

Если мы вызывем команду `git status`, то мы увидим все наши файлы, их состояние и подсказки Git, что мы можем с ними сделать. Какие файлы изменены, но не зафиксированы, то есть не закомичены

Какие статусы файлов есть в Git:

1. Не изменено __Unmodified__
2. Изменено `Modified`
3. Подготовленное _Staged_

## 6. Запись изменений в репозиторий

После инициализации репозитория, мы создаем файл, в котором мы дальше будем работать, писать нашу инструкцию. Для этого Git предлогает нам выполнить команду `git add` <имя нашего файла> для добавления нашего файла в репозиторий. Иными словами мы выполняем фиксацию наших файлов, чтобы можно их было коммитить.

После добавления файла, мы можем сделать свой первый коммит, для того чтобы сохранить наши данные в каталоге. Для этого можем воспользоваться следующими командами:

```Bash
git commit -m "здесь мы можем указать удобное для нас сообщение о наших действиях и изменениях"

Либо воспользоваться комбинированной командой, если позволяет нам Git

git commit -am "здесь мы можем указать удобное для нас сообщение о наших действиях и изменениях"

```

Если мы по работали и внесли изменения в наш проект, то перед следующим коммитом в наш проект, мы можем сравнить посмотрев тем самым разницу между коммитами, для этого мы воспользуемся командой `git diff`, то мы увидим, что изменилось в нашей работе и нашем проекте, и на сколько строк.


## 7. Просмотр истории коммитов

Чтобы увидеть нашу работу или вдруг работу нашего коллеги, который передал данный проект нам для доработки, можно посмотреть историю созданных коммитов, для этого мы воспользуемся командой `git log`. Эта команда одна из основных и наиболее мощных инструментов в Git. По умолчанию `git log` перечисляет коммиты, сделанные в репозитории в обратном к хронологическому порядке — последние коммиты находятся вверху. 

Можно разными способами выйти из просмотра логов `git log`:

1. ***HotKey Ctrl + C***
2. ***HotKey Ctrl + Break***
3. ***Ввести Q и нажать Enter***

При работе над проектом мы можем работать на разных ветках и после слияния веток, по работав на них, мы можем вызвать команду `git log --graph`, что бы увидеть историю нашу работу на разных ветках и как происходило слияние веток.

## 8. Игнорирование файлов 

Для того, чтобы исключить из отслеживания в репозитории определенные файлы или папки, необходимо там создать файл ***.gitignore*** и записать в него их названия или шаблоны, соответствующие таким файлам или папкам. В файле `.gitignore` можно хранить используемые картинки под их собственными названиями, то есть имея пять картинок, у нас получится пять строк с названием каждой картинки или если все картинки имеют одно и тоже расширение, например `.png`, тогда в самом файле `.gitignore` мы в одной строке пишет вот так  __*.png__, тогда все наши добавленные картинки с таким расширениям в дальнейшей работе будут игнорироваться автоматически.

## 9. Перемещение между сохранениями

Когда мы работаем и делаем сохранения в нашей работе, то есть фиксируем наши данные с помощью комманды `git commit -m`, то если мы вызовем команду `git log` то мы увидим все наши сохранения, например одно из таких сохранений выглядит так:

```Bash
commit 809d62066a9d47133fcadf41aeabf86c2675fe6f
```

И если мы введем первые четыре символа этого коммита с командой git checkout, например вот так: `git checkout 809d`, то мы сможем перемещаться между сохраненными коммитами.

## 10. Создание, слияние веток и разрешение конфликтов при слиянии веток в Git

По умолчанию имя основной ветки в Git - `master`

Создать новую ветку можно командой:

```Bash
git branch <имя новой ветки>
```

Для слияния выбранной ветки с текущей нужно выполнить команду:

```Bash
git merge <название выбранной ветки>
```

Для перехода с одной ветки на другую можно выполнить разными командами:

```Bash
git checkout <имя ветки на которую мы хотим перейти>
git switch <имя ветки на которую мы хотим перейти>
```
Для перехода с текущей ветки на новую и одновременно созданную, можно сделать так, выполнив команду: 

```Bash
git checkout -b <имя ветки на которую мы переходим и создаём одновременно>
```
Чтобы удалить ветку, нужно выполнить команду:

```Bash
gti branch -d <имя ветки, которую мы хотим удалить>
```
Но следует помнить, что нельзя удалить текущую ветку находясь в данный момент на ней, что бы удалить ту или иную ветку, нужно находиться в данный момент на любой другой ветки.

Для отмены крайнего слияния веток, нужно выполнить следующую команду:

```Bash
git merge --abort
```

Чтобы посмотреть список веток нужно выполнить команду:
```Bash
gti branch
```
 После вызова этой команды появится список всех имеющихся веток, а звездочкой будет отмечена та ветка, на которой мы находимся в данный момент.
 
 При выполнение команды слияния веток git merge <название выбранной ветки>, могут возникать конфликты, в некоторых случаях Git может самостоятельно решить проблему слияния, тогда конфликта не будет. Почему возникает конфликт при слиянии двух веток, если у нас совпадает информация на двух ветках, тогда Git предллогает нам право выбора, какое изменение принять и сохранить, предлогая нам очередную подсказку по изменениям и слиянию веток.
 
 ## 11. Работа с удаленными репозиториями
  
  1. Для того что бы начать нашу работу с удаленными репозиториями, нам нужно создать аккаунт на [сайте](https://github.com/), здесь у нас так же будут подсказки для регистрации в виде зеленых галочек, что озночает свободное имя для регистрации.
 
  2. Следующее что нам нужно сделать для дальнейшей работы, нужно создать локальный репозиторий. Мы уже умеем создавать локальные репозитории через команду __git init__, но в данном случае мы будем работать с чужим репозиторием, поэтому мы создаем новую папку, которая не под версионным контролем, после создания и открытия новой папки, мы переходим на наш зарегистрированный аккаунт на GitHub, через поисковую мы находим нужный нам репозиторий, нажимаем на него, следуя подсказкам Git, делаем Fork в правом верхнем углу над нашим репозиторием и тем самым копируем его в наш аккаунт на GitHub. После этого мы нажимаем зеленую кнопку с названием **`<>code`**, у нас открывается три графы, мы выбираем `HTTPS`, копируем нашу ссылку клавишей `copied`, тем самым наша скопированная ссыка будет выглядеть так:

```Bash

 https://github.com/RomanMolchanov777/SCV_GitPR.git

 ```
 3. Далее мы переходим в нашу созданную папку не под версионным контролем и переносим наш удаленный репозиторий в нашу папку командой:

 ```Bash
 git clone https://github.com/RomanMolchanov777/SCV_GitPR.git
 ```

 После этой команды наш удаленный репозиторий оказывается в нашей папке. Чтобы нам начать параллельно с ним в нашей папке, нам нужно сначала перейти в подпапку, для этого выполним следующую команду:

 ```Bash
 cd <указываем полное имя подпапки в которую нам нужно перерйти>
 ```

 4. Если мы создаем свой удаленный репозиторий, то нам нужно на GitHub через клавишу `+` создать новый репозиторий, при создании удаленного репозитория мы выбираем:

 * Название репозитория
 * Приватность репозитория
 * Можем указать комментарий к нашему репозиторию
 * Можем выбрать расширения для игнора файлов, папок и т.д.

 После этого мы создаем наш новый удаленный репозиторий

 5. Для добавления нашего удаленного репозитория в локальный мы копируем нашу ссылку на GitHub, как ранее мы это делали для клонирования и выполняем следующую команду:

 ```Bash
 git remote add <url - адрес нашего репозитория>
 ``` 
 Так же мы можем воспользоваться командой для детальной информации о нашем репозитории таким образом:

 ```Bash
 gti remote -v
 ```

Для принудительного переименования ветки `master` в `main`, выполняем команду:

```Bash
git branch -M <имя ветки>
```

Для того что бы отправить наш репозиторий после работы на GitHub нам нужно воспользоваться командой:

```Bash
git push -u <имя адреса для отправки репозитория>
```
Если вдруг нам после поправки нашего проекта на GitHub нужно вытянуть наш репозиторий обратно, мы можем воспользоваться командой:
```Bash
git pull
```
Тогда все наши изменения, которые были совершены на GitHub, снова вернуться в наш локальный репозиторий и мы детально будем видеть наши изменения
