# Инструкция по работе с GIT

## 1. Установка и настройка 

* Перейдите на [сайт](https://git-scm.com/download) и загрузите установщик.

Запустите установщик и следуйте инструкциям на экране. Вы можете оставить все параметры по умолчанию.

После завершения установки проверьте установку, открыв терминал и введя команду git --version. Это должно отобразить установленную версию Git.

* Настройка Git при первом использовании:

После установки настройте свое имя пользователя и адрес электронной почты, так как каждый коммит будет использовать эти данные.
* `git config --global user.name "Ваше Имя" `

* `git config --global user.email "ваша почта@example.com" `

## 2. Создание нового репозитория

* Клонирование репозитория на локальный компьютер:

Скопируйте URL-адрес репозитория и выполните команду:

`git clone url`

* Добавление файлов в локальный репозиторий:

Перейдите в каталог репозитория и создайте или переместите файлы, которые вы хотите добавить. Используйте команду git add для добавления файлов в индекс Git:

`git add .  # добавить все файлы`

`git add имя_файла  # добавить конкретный файл`

* Создание коммита:

Создайте коммит с помощью команды git commit:

`git commit -m "описание коммита"`

* Отправка изменений в удаленный репозиторий:

Используйте команду git push для отправки изменений:

`git push origin main`

* Просмотр статуса репозитория:

`git status`

Эта команда покажет вам, какие файлы были изменены, какие файлы еще не добавлены в индекс, и в какой ветке вы находитесь.

* Просмотр истории коммитов:

`git log`

Эта команда покажет историю коммитов для текущей ветки.

* Создание новой ветки:

`git branch имя_ветки`

* Переключение между ветками:

`git checkout имя_ветки`

* Слияние веток:

`git merge имя_ветки`

* Удаление файла из репозитория:

`git rm имя_файла`

* Откат изменений:

`git revert HEAD  # откатить последний коммит`

* Обновление локального репозитория до последней версии на сервере:

`git pull origin main`

## 3. Игнорирование фаилов 

* Создание файла .gitignore:

В корневой директории вашего репозитория Git создайте файл с именем **.gitignore**

Откройте файл .gitignore в редакторе.

* Синтаксис и правила файла .gitignore:

Игнорирование конкретных файлов:

Укажите имя и расширение файла, например:

`example.txt`

* Игнорирование всех файлов с определенным расширением:

Используйте символ * вместе с расширением файла, например:

`*.log`

* Игнорирование конкретных папок:

Укажите путь к папке с закрывающим слешем, например:

`logs/`

* Игнорирование всех файлов в определенной папке и во всех ее подпапках:

Используйте следующий синтаксис:

`logs/**/*`

* Игнорирование всех файлов, кроме определенных:

Используйте символ `!` перед именем файла, который не следует игнорировать, например:

`*
!important.log`

В данном примере все файлы будут игнорироваться, кроме important.log.

Любая строка, начинающаяся с символа #, будет рассматриваться как комментарий и игнорироваться, например:

`# Это комментарий`

* Сохранение и коммит файла .gitignore:

Сохраните изменения в файле .gitignore.
В командной строке или терминале добавьте файл .gitignore в ваш репозиторий Git, используя следующую команду:

`git add .gitignore`

Сделайте коммит, чтобы сохранить файл .gitignore в вашем репозитории, используя следующую команду:

`git commit -m "Add .gitignore file"`

## 4. Разрешение конфликтов

Разрешение конфликтов в Git - важный процесс, который помогает объединить изменения из разных веток в одну. Конфликты часто возникают в результате слияния (merge) или вытягивания (pull) изменений. 

* Определение конфликтов:

Первым делом, определите, в каких файлах возникли конфликты. Git явно указывает на конфликтующие файлы в выводе команды слияния или вытягивания.

`git status`

* Открытие конфликтующих файлов:

Откройте конфликтующие файлы в вашем редакторе или IDE. Git помечает конфликтные области следующим образом:

```
<<<<<<< HEAD
ваши локальные изменения
=======
изменения из ветки, с которой вы сливаетесь
>>>>>>> branch-name
```
* Разрешение конфликтов:

Редактируйте файлы, чтобы исправить конфликты. Выберите нужные строки кода, и удалите метки конфликта (<<<<<<<, =======, >>>>>>>).

* Проверка решения:

После разрешения всех конфликтов, проверьте работу вашего кода, запустив его и проведя необходимые тесты.

* Добавление и коммит изменений:

Добавьте измененные файлы в индекс Git с помощью команды git add:


`git add имя_файла`
`# или добавьте все файлы: git add .`

Теперь создайте новый коммит, чтобы сохранить ваши изменения:

`git commit -m "Разрешение конфликтов во время слияния"`

* Продолжение процесса слияния:

Если вы были в процессе слияния, теперь его можно завершить. Если вы были в процессе вытягивания изменений, теперь у вас должна быть последняя версия кода.

* Проверка истории коммитов (опционально):

Проверьте историю коммитов, чтобы удостовериться, что все изменения корректно зафиксированы:

`git log --oneline --graph --all`

# Работа с удаленным репозиторием

1. Создать акаунт на GitHub

* Переход на сайт GitHub:

Откройте веб-браузер и перейдите на сайт GitHub.

* Регистрация:

На главной странице GitHub введите свой адрес электронной почты в предлагаемое поле, затем нажмите кнопку "Sign up for GitHub".

2.  Отправка изменений в удаленный репозиторий:

* Сначала создайте коммит:
  
`git add .`
`git commit -m "Описание изменений"`

* Отправьте коммит в удаленный репозиторий:

`git push origin main`

3. Получение изменений из удаленного репозитория:
   
* Чтобы загрузить последние изменения без объединения:

`git fetch origin`

* Чтобы загрузить и объединить изменения:

`git pull origin main`

4. Отправка и получение веток:

* Отправьте новую ветку в удаленный репозиторий:

`git push origin имя_ветки`

Получите ветку из удаленного репозитория:

`git fetch origin имя_ветки:имя_ветки`

5. Просмотр списка удаленных репозиториев:

Чтобы посмотреть список удаленных репозиториев и их URL-адреса, используйте:

`git remote -v`

6. Pull request

Для перехода в клонированный (командой clone) репозиторий необходимо выполнить команду:

`cd <имя_файла.git>`

Для внесения изменений в клонированный файл создаем новую ветку и переходим на нее. Создаем новый отслеживаемый файл имя_файла.md и заполняем его информацией и фиксируем изменения командами:

`git add имя_файла.md`
`git commit -m "message"`

Отправляем измененный файл на Github командой

`git push`
