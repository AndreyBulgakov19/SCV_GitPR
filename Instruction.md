# Работа с Git
![GIT](2.png)

## 1.Проверка наличия установленного Git
В терминале выполнить команду 
```
git --version
```
  Если **Git** не установлен, то появится сообщение об ошибке.

## 2.Установка Git
Загружаем последнюю версию **Git** с [сайта](http://git-scm.com/downloads). 
Устанавливаем с настройками по умолчанию.

## 3.Настройка Git
При первом использовании необходимо представиться. Для этого нужно ввести 2 
команды:
```
git config --global user.name <ваше имя>
git config --global user.email <ваша почта>
```

## 4.Инициализация репозитория
Для того, чтобы начать слежение за существующим проектом надо перейти в соответствующую папку и выполнить следующую команду, чтобы создать репозиторий (хранилище файлов):
```
git init
```
Т.о. в исходной папке появится скрытая папка ***.git***
## 5. Начало установления контроля за изменениями файлов.
Чтобы начать контроль за версиями существующих файлов введите команду:
```
git add <имя файла>
```
Эта команда позволяет начать слежение за новыми файлами и производить индексирование файлов (хэш).

Затем надо ввести команду commit (совершать, фиксировать) для сохранения изменений в файле с комментариями этих изменений.
 ```
git commit -m "комментарий"
 ```
Если не написать сообщение коммита, то коммит не применится.
Обобщенная команда, включающая сразу 2 команды:
```
git commit -am "комментарий"
```
 
 Вернуть файл в состояние последнего коммита:

```
git restore <name>
```
## 6. Просмотр статуса Git
Выполните команду просмотра о состоянии рабочего каталога в полной или короткой версии этого отчета:
```
git status
git status -s
git status --short
```
## 7. Просмотр индексированных и неиндексированных изменений.
Выполните следующую команду, которая показывает разницу между текущим файлом и закоммиченным файлом:
```
git diff    
```
## 8. История изменений коммитов
Просмотр истории создания коммитов, начиная с самого последнего по ветке , где находитесь, иначе надо указать название ветки:
```
git log
git log --oneline
git log --graph
git log --oneline --decorate --graph --all
```
История изменений по всем веткам:
```
git log --all --oneline
```
Выйти из выполнения этой команды, если вывод слишком длинный:
```
Ввести Q и нажать Enter
```
Следующая команда позволяет переключаться между коммитами:
```
git checkout <хэш>
```
## 9. Просмотр истории всех операций
Просмотр истории всех операций (в том числе удаление, отмена, перехода по истории, создание новых обьектов)
```
git reflog
```
## 10.Создание ветки          
По умолчанию первая созданная ветка  __*master*__, ее можно переименовать.

Список всех веток в репозитории можно просмотреть с помощью команды:
```
git branch
```
Текущая ветка будет отмечена  *

Последний коммит, выполненный в каждой ветке, демонстрирует команда 
```
git branch -v
```

Создать новую ветку и сразу переключиться на неё с помощью флага __*-с*__ или __*-b*__:
```
git switch -c <name branch>
git checkout -b <name branch>
```
## 11.Объединение веток

Для объединения веток надо перейти сначала в __*master*__ ветку, затем выполнить команду:
```
git merge <name branch>
```
Режим __*Fast forward*__: перемотка вперед (перемещение вперед по записи).
Указатель __*HEAD master*__ перемещается вперед на место указателя __*HEAD*__ объединяемой ветки линейно, если в ветке __*master*__ не было выполнено никаких изменений.
Если в ветке __*master*__ делать какие-то изменения, то тогда будет ветвление.

Если была изменена одна и та же часть файла в обеих ветках, то может возникнуть конфликт, который потребует участия пользователя. __*VSC*__ предлагает варианты разрешения, нужно выбрать один из вариантов. Обязательно коммит слияния.

## 12.Cлияние, удаление веток и разрешение конфликта.

Для слияния ветки с текущей нужно выполнить команду:

```
git merge <name branch>
```
Для удаления ветки надо предварительно перейти в другую ветку, иначе операция не будет выполнена:
```
git branch --delete <name branch>
git branch -d <name branch>
```
Если была изменена одна и та же часть файла в обеих ветках, то может возникнуть конфликт, который потребует участия пользователя.
__*VSC*__ предлагает варианты разрешения, нужно выбрать один из вариантов. Обязателен коммит слияния.
Если ветка не слита, но ее все равно можно принудительно удалить с помощью команды:
```
git branch -D <name branch>
```
## 13. Перемещение между ветками

Следующая команда используется для переключения веток и выгрузки их содержимого в рабочий каталог.
```
git checkout <name branch>
git switch branch <name branch>
``` 
Вернуться к предыдущей ветке: 
```
git switch -
```
Вернуться к актуальному состоянию и продолжить работу:
```
git checkout master
``` 
## 14.Исключение файлов из отслеживания (игнорирование):

Создать в папке файл _**.gitignore**_ 

Записать в него шаблон из набора символов, содержащихся в имени. Т.о.такие файлы будут игнорироваться, например: __*.png*___

Затем выполнить команды:
```
git add .gitignore
git commit -m "сообщение"
git status
```
Теперь файл _**2.png**_ не выводится в распечатке статуса и не отслеживается, отображение его имени стало серого цвета.
## 15. Работа со своим удаленным репозиторием.
* Создать аккаунт на сайте _**github.com**_
* Создать репозиторий: нажать + в соответствующем поле, ввести имя, нажать кнопку __*Create repository*__
* Подружить свой локальный и удаленный репозиторий:
```
git remote add origin https://github.com/Albina7333331/TEST.git
```
* На своем локальном компьютере создать репозиторий, внести изменения в файл, закомитить.
* Отправить файл в свой репозиторий
```
git push -u origin main
```
где
__*origin*__ -имя удаленного репозитория (по умолчанию)
__*main*__ - имя ветки (по умолчанию)


* Для обратной операции: сделали изменения на GitHub в своем репозитории, создали коммит. Затем на своем локальном компьютере выполнили следующую команду:
```
git pull
```
Т.о. изменения появятся на локальном компьютере.

## 16.Работа с чужими репозиториями (**pull request**).

* Для создания копии чужого репозитория в своём аккаунте на GitHub зайти, например, на чужой аккаунт по ссылке **"AndreyBulgakov19/SCV_GitPR"** и с помощью кнопки **"Fork"** выполнить копирование выбранного репозитория в свой аккаунт.
Произойдет копирование и откроется страница своего аккаунта.

* Затем на своем локальном компьютере выполнить клонирование этого репозитория со своего аккаунта:
```
git clone https://github.com/Albina7333331/SCV_GitPR.git
```
* Создать новую ветку __*new*__ для работы с файлом (черновик) и сразу перейти в неё:
```
git branch -b new
```
* Отредактировать файл, создать коммит и отправить в свой удаленный репозиторий на GitHub:
```
git push --set-upstream origin new
git push -u origin new
```

В своем репозитории на GitHub обновить страницу, появится новая ветка __*new*__

* Зафиксировать изменения (коммиты).

* На сайте GitHub нажать кнопку **Compare&Pull request**.
Оставить комментарий.

* В чужом репозитории появится активная кнопка **Create Pull request**
