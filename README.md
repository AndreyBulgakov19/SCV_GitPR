# Работа с Git
## 1. Проверка наличия установленного Git
в терминале выполнить команду ` git --version `. Если Git установлен, появиться сообщение с информацией о версии программы, иначе будут сообщение об ошибке.

## 2. Установка Git
Загружаем последднюю версию Git с [сайта](https://git-scm.com/downloads)
Устанавливаем с настройками по умолчанию.   

## 3. Настройка Git
При первом испольщовании Git необходимо предстваиться. Для этого нужно ввести в терминале 2 команды:
```Bash
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```

## 4. Иницилизация репозитория.
В терминале переходим к папке
, в которой хотим создать репозиторий. Выполняем команду:
```
git init
```
В исходной папке появится скрытаяя папка ***.git***

## 5. Основные команды Git.
1. `git init`- инициализация локального репозитория;
2. `git status`- получить информацию от git о его текушем состоянии;
3. `git add`- добавить файл или файлы к следующему коммиту;
4. `git commit -m(message)`- создание коммита. ***обязательно нужно использовать`-m` воизбежании попадения в консоль.***;
5. `git log`- вывод на экран истории всех коммитов с их хеш-кодами;
6. `git log --oneline`- вывод на экран сокращенный вариант истории всех коммитов с первыми 7 символами хеш-кода;
7. `git checkout`- переход от одного коммита к другому;
8. `git chekout master`- вернуться к акутуальному состоянию и продолжить работу;
9. `git diff`- позволяет увидеть разницу между текущим файлом и закоммиченным файлом:
10. `git branch`- выводит список веток в репозитории
+ `git branch name`(где name- имя)- создание новой ветки с именем.
+ `git branch -d name`-удаление ветки с выбранным именем. 
+ `git checkout branch name `- переход с одной ветки на другую. 
+ `git log --graph`- вывод списка коммитов в виде красивого дерева.
+ `git merge name`- слияние веток(слив ветки идет туда где ты находишься, поэтому спосле сохранения данной ветки необходимо перейти на ту, куда должно слиться(например сохраняю ветку text_comand, перехожу на ветку master и только потом сливаю в master))**Возможны конфликты!!!**
+ `git commit --amend -(a)m`(а если надо сохранить) данная команда помогает удалить прошлый коммит и исправить данный. 
## 6. Консоль
При попападании в консоль:

1. **В консоли не работает мышь, только клавиатура!**;
2. Нужно перейти на английскую расскладку и нажать букву `I`;
3. В самой нижней строке будет написанно `ВСТАВКА`, после чего можно добавить комментарий;
4. После ввода комментария, необходимо нажать `Esc`;
5. Как только пропадет из нижней строки слово `ВСТАВКА`, нужно написать `:wq` (данная команда является сокращением от `write` и `quit`) и нажать `Enter`.
 
## 7. Игнорирование файлов 
Для того чтобы исключить из отслеживания в репозитории определенные файлы или папки, необходимо создать там файл `.gitignore` и записать в него их названия или шаблоны, соответствующие таким файлам или папкам.  

# 8. Работа со слиянием веток
## 1.Создаем репозиторий 
* Создаем папку и открываем ее через VS Code;
* Введя команду `git status` в терминале, убеждаемся что мынаходимся просто в папке, и что мы готовы создать репозиторий;   
* Через команду `git init` создаем репозиторий;
## 2. Создание веток
* Для просмотра всех соществующих веток на даный момент, вводим команду `git branch`;
* Для создания новой ветки используем эту же командку с приставкой ее имя (`git branch second_branch`); 

## 3.Слияние веток 
* После создания новой ветки и добавления в нее информации, необходимо сохранить данные, используя команду `git add` и после `git commit ` либо же использовать комбинированную команду `git commit -am`  
* Убедившись, что все сохранено, переходим в ветку, в которую все будет сливаться
* Если в данной ветке не было изменений, то сразу переходим к слиянию(в противном случае, неоюходимо сначала сохранить изменения и после  приступить к слиянию )
* С помощью команды `git merge (название ветки)` производим слияние веток, после успешно сохраняем полученный результат. ***Могут быть конфликты*** Если информация которая была в данной ветке и информация из ветки которая сливается, как-то пересекаются, **возникнет конфликт**.   Такое бывает, необходимо выбрать, как лучше сохранить информацию 
## 4. Схема дерева веток 
+ Для того чтобы увидеть наглядную схему дерева веток можно использоваь команду `git log --graph `. Данная команда выводит схему всех веток, показывая ветвление и слияния их.
+ Также можно использовать команду `git log` для вывода всех коммитов с их хеш-кодами. Есть сокращенный вариант данной команды. `git log --oneline` так же показывает все коммиты, но в сокращенном варианте с первыми 7 символами хеш-кода.

# 9. Основной синтаксис языка `Markdown`
 №1. ***Перенос:***  
+ с помощью тега `br`(в любом месте) (его нужно взять в треугольные скобки<>);\
+ с помощью обратного слеша (в конце строки)  `\`;  
+ с помощью двойного пробела.  

№2. ***Заголовки:***  
Ставиться знак `#` в начале строки, количестов этих знаков уменьшает уровень заголовка:

# Заголовок первого уровня
## Заголовок второго уровня ##
### Заголовок третьего уровня
#### Заголовок четвёртого уровня #
##### Заголовок пятого уровня ############
###### Заголовок шестого уровня <hr>

\
№3. ***Выделение текста:***
+ *Курсив* (необходимо выделить слово с обеих сторон `* *` или `_ _ `)
+ **Жирный**  (необходимо выделить слово с обеих сторон `** **` или `__ __ `)
+ ***Жирный курсив*** (необходимо выделить слово с обеих сторон `*** ***` или `___ ___ `)
+  кор*рек*тно, кор**рек**тно, кор***рек***тно(можно также использовать знак `*` внутри слова)
+ ~~Зачёркнутый текст~~ (Чтобы зачеркнуть текст, нужно использовать две тильды `~~`) 


№4. **Цитаты:**
> Оформление цитатой
последовательных строк
внутри одного параграфа  

(Чтобы параграф отобразился как цитата, нужно поставить перед ним закрывающую угловую скобку `>`.)
***

> # Заголовок
> Первый параграф
>
> Второй параграф
>
> > Вложенная цитата
> > > Цитата третьего уровня
>
> Продолжение основной цитаты

Возможна также уровневая градация.
***
№5. **Разделители:**\
Чтобы оформить горизонтальный разделитель, нужно поставить три или больше специальных символа: звёздочки *, дефиса - или нижних подчёркивания _. Они должны находиться на отдельной строке, и между ними можно ставить любое количество пробелов и табуляций.

Если ваш редактор поддерживает HTML-теги, то для разметки можно также использовать тег `<hr>`.
***
№6. **Списки:**
* Ненумерованные (любой из символов `*` `-` `+`)

* Нумерованные (`1.` `2.` `3.` );

* Вложенные (Чтобы создать вложенный список, нужно поставить перед его пунктами табуляцию или несколько пробелов. В Markdown одна табуляция соответствует четырём пробелам.)  

1. Пункт
	1. Подпункт
		1. Подподпункт

- Пункт
	- Подпункт
		- Подподпункт


1. Пункт
	- Подпункт
		* Подподпункт

+ Пункт
	1. Подпункт

№7. **Ссылки**(необходимо заключить её в угловые скобки.)\
Чтобы оформить ссылкой часть текста, используется такой синтаксис: `[текст](ссылка)`.

№8. **Вставка кода**  
Нужно с двух сторон выделить этот код одним или несколькими обратными апострофами `  
```
#include <stdio.h>
int main() {
   printf("Hello, World!");
   return 0;
}
```
(Если обрамлять код тремя обратными апострофами, то после первой тройки можно указать язык программирования — тогда Markdown правильно подсветит элементы кода.)
```python
if x > 0:
	print (x)
else:
	print ('Hello, World!')
```

№9. **Как использовать Markdown в мессенджерах**  

* Жирный:

**Telegram и Discord** — `**две звёздочки с двух сторон**;`
WhatsApp и Viber — *одна звёздочка с двух сторон*.

* Курсив:

**Telegram** — `__два нижних подчёркивания с двух сторон__;`
**WhatsApp и Viber** — `_одно нижнее подчёркивание с двух сторон_;`
**Discord** — `*одна звёздочка с двух сторон* или _одно нижнее подчёркивание с двух сторон_.`

* Подчёркнутый:

**Discord** — `__два нижних подчёркивания с двух сторон__.`

* Зачёркнутый:

**WhatsApp и Viber** — `~одна тильда с двух сторон~;`  
 **Discord** — `~~две тильды с двух сторон~~.`

* Моноширинный (используется для вставки кода):

**Telegram, WhatsApp, Viber и Discord** —` ```три обратных апострофа с двух сторон```;`

в Discord точно так же, как и в Markdown, можно указывать язык программирования для подсветки синтаксиса.

* Спойлер:

**Telegram и Discord** — `||две вертикальные черты с двух сторон||.`  
№10. **Добавление картинок**
1. Необходимо поместить файл изображения в папку, где создан репозиторий
2. Следом нужно ввести команду `![]()`- в квадратных скобках нужно указать информацию в случае отсутсвия картинки, в круглых скобках надо написать названия картинки с расширением. Например `![здесь фотка бомбочек] (bombochka.jpg)` ![Здесь фотка босбочек](kak-sdelat-bombochku-dlya-vanny-3.jp)   
или   <hr>
 ![Здесь фотка босбочек](kak-sdelat-bombochku-dlya-vanny-3.jpg) 
После добавления картинки в папку, git при команде `git status` будет показывать не наблюдаемый файл той самой картинки, ее сохранять не нужно, но для того чтобы она не мешалась, ее можно переместить в `ignore`. Для этого необходимо:
* Создать файл ***.gitignore*** в проводнике VS Code(или другой программмы) 
* После создания файла достаточно всего лишь вписать название с расширением не нужного файла, и после сохранения текста, git будет игнорировать файл.(сам файл `.gitignore` тоже необходимо сохранить).
# 10. Работа с удаленными репозиториями 
* Для начала необходимо зарегистрироваться на GitHub
* Необходимо создать пустую папку, в которую будем все копировать
* После регистрации необходимо "подружить" VS Code и GitHub(необходимо при первом клонировании ввести данные Github и тогда они "подружаться")
* Как только мы их подружили, можно начинать клонирование с помощью команды `git clone ссылка_на_удаленный_репозиторий` 
* После клонирования репозитория, необходимо переместиться в сам репозиторий, либо через открытие в новом терменале, либо через команды в текущем терминале
* Успешно выполнив слияние и перейдя в рабочий репозитории можно начинать работу(предварительно создать отдельную ветку)
* Как только мы закончим добаление всех информации, предварительно закоммитив репозиторий, можно преступить к "сливу" информации в удаленный репозиторий. Это можно выполнить с помощью команды `git push -u origin имя_ветки`(Для этого необходимо провести представление (см. п.3 в первой главе ))
* Когда мы слили все в удаленный репозиторий, надо зайти на Github и нажать зеленую кнопку `Compare and pull request`.
