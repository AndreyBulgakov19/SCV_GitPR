# Работа с Git
## 1. Проверка наличия установленного Git
В терминале выполнить команду 
```
git --version
```

 Если Git установлен появится сообщение с информацией о версии программы, иначе будет сообщение об ошибке.

## 2. Установка Git
Загружаем последнюю версию программы с [сайта](https://git-scm.com/downloads). Устанавливаем с настройками по умолчанию.
## 3. Настройка  Git
После установки необходимо «представиться» системе контроля версий. Это нужно сделать всего один раз, и git запомнит вас. Для этого нужно ввести в терминале 2 команды:

* git config --global user.name «Ваше имя английскими буквами»

* git config --global user.email ваша почта@example.com

## 4. Инициализация репозитория 
В терминале выполнить команду:
```
git init
```
В результате в существующей папке появится еще одна папка с именем **.git** и всеми нужными вам файлами репозитория — это будет основа вашего Git-репозитория.
Пока контроль ваших файлов отсутствует.


## 5. Запись изменений в репозитории
Основным инструментом определения состояния файлов является команда:
```
git status
```
Показывает текущее состояние гита, есть
ли изменения, которые нужно закоммитить
(сохранить).

 **_Чтобы вызвать ранее введённую команду,
пользуемся стрелками на клавиатуре.
Перебираем недавно введённые команды
нажатием стрелки «вверх»_.**

 После вызова команды git status, Git говорит:

* есть не отслеживаемые файлы — Untracked files;

* файлы появились, но он за ними пока не следит.

 * нет изменений:
![Гитстатус](Статус.jpg)

Если хотите, чтобы Git начал отслеживать изменения в этих файлах, их надо добавить. Введите команду и укажите название файла, чтобы включить его в фиксацию:
```
git add
```

*Писать название файла целиком не обязательно: терминал дозаполнит данные автоматически.*

Теперь можно сохранить те изменения, которые были совершены. Если надо сделать некоторые сохранения или зафиксировать
текущий статус, вызываем команду:
```
git commit
```
Чтобы в дальнейшем понимать, какие были сохранения, можно оставить комментарий: к команде commit добавим параметр m — сокращение от message («сообщение») — и в кавычках запишем то, что
надо запомнить, некий комментарий к сохранению.

## 6. Просмотр истории коммитов

Для просмотра журнала изменений введем команду:
```
git log
```
По умолчанию команда git log выводит в обратном
хронологическом порядке список сохраненных в данный репозиторий версий. То
есть первыми показываются самые свежие коммиты.
## 7. Перемещение между сохранениями (коммитами)
Чтобы перейти к какой-то версии, сохранению, необходимо
вызвать команду:
```
git checkout
```
После просмотра интересующего коммита, необходимо вернуться в тот, где работаете сейчас, вызвав команду:
```
git checkout master
```

*Git сравнивает записанные версии файлов. Если что-то меняете, но
изменения не записали, для Git файл не изменится.*


Для того, чтобы не упустить и записать изменения, можно посмотреть разницу между текущей и уже зафиксируемой версией файла, вызвав команду:
```
git diff
```
## 8. Игнорирование файлов
Для того, чтобы исключить из отслеживания в репозитории определенные файлы или папки необходимо создать файл ***.gitignore***  и записать в него их названия или шаблоны, соотвествующие таким файлам или папкам.

## 9. Создание веток в Git
По умолчанию имя основной ветки в Git - **master**.
Создать ветку можно командой:
~~~bash
git branch <имя новой ветки>
~~~

Список веток в репозитории можно посмотреть с помощью команды:
~~~bash
git branch 
~~~
Обратите внимание на символ *, стоящий перед веткой master: он указывает на ветку, на которой вы находитесь в настоящий момент:

![Скрин](СписокВеток.png)



Для того, чтобы перейти на другую ветку, необходимо ввести команду:

~~~
git checkout <имя новой ветки>
~~~

## 10. Слияние веток и разрешение конфликтов
Для слияния веток необходимо перейти на ветку ***в которую необходимо перенести данные*** и далее ввести команду:
~~~bash
git merge <имя ветки из которой переносим данные>
~~~


Если данные на ветках различаются, то возникает конфликт, необходимо выбрать одну из предложенных команд:
![Скрин](КонфликтСлияния.png)


После конфликта необходимо сделать конфликт слияния (зафиксировать слияние) с помощью команды “git commit”, можно также отменить слияние командой “git merge –abort”


## 11. Удаление веток
Для того, чтобы удалить уже слитую ветку необходимо вызвать команду:

~~~bash
git branch -d <название ветки>
~~~

Удалять ветку в которой мы находимся нельзя.
 ##   12. Работа с удалёнными репозиториями
 1. Создать аккаунт на GitHub
 2. Создать локальный репозиторий
 3. Создать удалённый репозиторий.
 В правом углу главной страницы нажать "+", далее выбрать "Новый репозиторий".
 Откроется окно создания: 
заполнить имя репозитория - название файла в гит и далее по пунктам.
После создания репозитория,  GitHub предложит действия (подсказки) для нового репозитория:
~~~bash
echo "# remote" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/Name/remote.git
git push -u origin main 
~~~


или действующего репозитория:

~~~bash
git remote add origin https://github.com/Name/remote.git
git branch -M main
git push -u origin main
~~~

 4. Связать локальный репозиторий с удалённым.

Вводим команду:
~~~bash
git remote add origin https://github.com/Name/remote.git
~~~

Чтобы проверить есть ли привязка пользуемся командой:

~~~bash
git remote
~~~

Появится весь список удаленных репозиториев привязанных к локальному.

Более подробный список с адресами можно вызвать командой: 

~~~bash
git remote -v
~~~

Далее необходимо в git *переименовать* ветку main, потому что на GitHub основная ветка main (помогает избежать путаницы), далее перейти в нужную ветку и вызвать команду:

~~~bash
git branch -M name(новое имя ветки)
~~~

 5. Сделать ветку активной, для отслеживания.

 Вводим команду, которая позволяет отправить нашу версию репозитория на внешний
репозиторий: 

~~~bash
git push
~~~

При обновлении страницы на GitHub, должен появится репозиторий.

6. Редактирование удаленного репозитория

Для редактирования можно выбрать кнопку "карандаш" -  октроется режим редактирования.
Будут доступны две кнопки: Edit (редактирование) и  Preview (отображение файла).

Необходимо фиксировать изменения, выбрав кнопку "Commit message".

Для того, чтобы получить изменения из удаленного репозитория и выполнить слияние с локальной версией:

~~~bash
git pull
~~~

Отправить локальную версию репозитория на внешний `git push`



Команда позволяет склонировать внешний репозиторий на наш ПК:

 ~~~bash
 git clone
 ~~~~



***Как сделать pull request:***

* Делаем fork репозитория

* Делаем clone СВОЕЙ версии репозитория
* Создаем новую ветку и в НЕЕ вносим свои изменения

* Фиксируем изменения (делаем коммиты)

* Отправляем свою версию в свой GitHub

* На сайте GitHub нажимаем кнопку pull request
