# Инструкция по работе с Git
## 1. Что такое Git?
Git - это одна из реализаций распределенных систем контроля версий, имеющая как локальные, так и удаленные репозитории. Яаляется самой популярной реализацией систем контроля версий в мире.
## 2. Установка наличия установленного Git
В терминале выполнить команду `Git --version` . Если Git установлен, появится сообщение с информацией о версии программы, иначе будет сообщение об ошибке.
## 3. Установка Git
Загружаем последнюю версию Git с [сайта](https://git-scm.com/downloads). Устанавливаем с настройками по умолчанию.
## 4. Настройка Git
При первом использовании Git необходимо представиться. Для этого нужно ввести в терминале две команды:
```Bash
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```
## 5. Подготовка репозитория
Для создания нового репозитория необходимо выполнить команду `git init` в папке с репозиторием, и у вас создаться репозиторий (появиться скрытая папка .git).
## 6. Создание комитов
### Git add
Для добавления изменений в коммит используется команда `git add` . Для этого выполняется команда:
```Bash
git add <название файла>
```
### Git status
Для того, чтобы посмотреть текущее состояние репозитория используется команда `git status` . С ее помощью можно увидеть были ли изменения в файлах, или их не было. 
### Git commit
Для того, чтобы создать коммит (сохранить или зафиксировать) необходимо выполнить команду `git commit` . Выполняется она так:
```Bash
git commit -m <Текст сообщения>
```
Вместо параментра -m, можно использовать комбинацию параментров -am. Эта комбинация параментров создает коммит всех проиндексированных изменений и добавляет к коммиту подставленный комментарий.
### Git diff
Для того, чтобы увидеть разницу между двумя коммитами (текущим и сохраненным), используется команда `git diff` (с указанием промежутка между коммитами).
## 7. Просмотр истории коммитов
Для того, чтобы посмотреть журнал изменений истории коммитов используется команда `git log` . В ней содержится вся информация о каждом отдельном коммите, с указанием его хэша, автора, списка изменений и даты, когда они были сделаны.
Если в конце истории коммитов появляется END, то надо нажать клавищу *q*. Она возвращает в исходное окно ткрминала.
Команда:
```Bash
git log --graph
```
также позволяет просматривать историю с отображением граф с ветвлениями и историей слияний.
## 8. Перемещение между сохранениями
В Git под термином *checkout* подразумевают переключение между различными версиями объекта.
Команда `git checkout` работает с тремя различными объектами: файлами, коммитами и веткми.
Если мы хотим переместиться между ветками, то выполняется команда:
```Bash
git checkout <название ветки>
```
Еше одна команда, которая используется для переключения между ветками `git switch` . Выполняется она так:
```Bash
git switch <название ветки>
```
Чтобы вернуться в тот коммит, где мы работаем, т.е. к текущему состоянию проекта, используется команда:
```Bash
git checkout main
```
При этом *main* - это главная ветка по умолчанию.
## 9. Игнорирование файлов
Для того чтобы исключить из отслеживания в репозитории определенные файлы или папки, необходимо создать файл ***.gitignore***. В данный файл записываются вручную все названия файлов или папок, соответствующие таким файлам или папкам.
Репозиторий может иметь один файл ***.gitignore*** в корневом каталоге, который применяется ко всему репозиторию.
Каждая строка в ***.gitignore*** файле определяет шаблон. Git обычно проверяет в ***.gitignore*** файле шаблоны из нескольких источников от самого высокого до самого низкого. Например:
Шаблоны считываются из командной строки для тех команд, которые их поддерживают.
Шаблоны считываются из ***.gitignore*** файла в том же каталоге, что и путь, или в любом родительском каталоге, при этом шаблоны в файлах более высокого уровня переопределяются шаблонами в файлах более низкого уровня вплоть до каталога, содержащий файл. 
Например, формат шаблона:
Если пустая строка не соответствует ни одному файлу, то ее можно использовать в качестве разделителя для удобства чтения.
Строка, начинающаяся с "#", служит комментарием.
Косая черта "/" используется в качестве разделителя каталогов.
Звездочка "*" соответствует всему, кроме косой черты.
Символ "?" соответствует любому символу, кроме "/". 
Две последовательные звездочки "**" в шаблонах, соответствующие их полному пути, могут иметь разные значение.
## 10. Создание веток в Git
По умолчанию имя основной ветки в Git - `main` .
Создать ветку можно командой:
```Bash
git branch <название новой ветки>
```
Здесь команда `git branch` только создает новую ветку, но не переключается на нее. Чтобы переключиться на новую ветку выполняем команду:
```Bash
git checkout <название созданной ветки>
```
Создать новую ветку и переключиться на нее можно с помощью команды:
```Bash
git switch -c <название ветки>
```
Чтобы создать ветку и сразу же на нее переключиться, используется команда:
```Bash
git checkout -b <название ветки>
```
## 11. Слияние веток и разрешение конфликтов
Для слияния выбранной ветки с текущей нужно выполнить команду:
```Bash
git merge <название выбранной ветки>
```
Если была изменена одна и таже часть файла в обеих ветках, то может возникнуть конфликт, который потребует участия пользователя.
Если Git столкнется с конфликтом во время слияния, она отредактирует содержимое файлов с помощью визуальных маркеров:
```Bash
<<<<<<< , ======= , >>>>>>>
```
Эти маркеры показывают, где необходимо урегулировать конфликт.
После обнаружения конфликтующих участков кода вы можете исправить их по своему усмотрению.
VSC предлагает вариант разрешения этих конфликтов:
    1. Принять текущее изменения.
    2. Принять входящие изменения.
    3. Принять оба изменения.
    4. Сравнить изменения.
    5. Разрешить в редакторе слияния.
![Skrin](Skrin.png)
После принятия изменений и разрешения конфликта необходимо выполнить коммит слияния с помощью команды:
```Bash
git commit -am "Разрешение конфликта"
```
## 12. Удаление веток
Для того, чтобы удалить ветку используют команду `git branch` .
Выполняется она так:
```Bash
git branch <нвзвание выбранной ветки>
```
Git не позволяет удалить ветку, если в ней есть неслитые изменения. При попытке удалить неслитую ветку команда выдаст сообщение об ошибке.
Если вы действительно хотите удалить ветку, Git предлагает удалить ветку принудительно, даже если в ней есть неслитые изменения:
```Bash
git branch -D <название ветки>
```
Эта команда удаляет ветку независимо от ее состояния и не выдает никаких предупреждений.
Также нельзя удалить ветку если вы находитесь в самой этой ветки.
## 13. Работа с удаленными репозиториями
Локальным является репозиторий, созданный на своем компьютере.
Удаленный репозиторий - это версия, например, моего или чужого проекта, сохраненная на удаленном сервире.
Для того, чтобы загрузить данные с удаленного репозитория, сначала к нему нужно подключиться.
Для этого:
    1. Создаем аккаунт на GitHub.
    2. Создаем локальный репозиторий в VSCode.
    3. Создаем удаленный репозиторий.
Для того, чтобы создать свой репозиторий на GitHub, необходимо на главной странице своего аккаутна нажать на зеленую кнопку `New`. Открывается страница создания репозитория, где в поле `Repository name` указывается имя репозитория. Для завершения процесса создания репозитория нажимаем на кнопку `Create repository`.
    4. Связываем удаленный репозиторий.
Для добавления чужого репозитория в свой аккаунт на GitHub используется кнопка `Fork` на странице чужого репозитория. `Fork` позволяет скопировать чужой репозиторий в свой аккаунт в GitHub.
Для того чтобы добавить удаленный репозиторий к локальному, используется команда
```Bash
git remote add origin <ссылка на удаленный репозиторий>
```
Ссылка на удаленный репозиторий берется в GitHub, нажав на зеленую кнопку `Code` на странице удаленного репозитория. Нам предложат выбрать одну из трех ссылок. Мы выбираем ссылку на `HTTPS-ссылка`.
Еще одним вариантом является клонирование удаленного репозитория к себе на компьютер. Для этого используеи команду
```Bash
git clone <ссылка на удаленный репозиторий>
```
Операция клонирования создает на нашем компьютере точную копию удаленного репозитория.
### Git pull
Для того чтобы получить изменения и сразу обновить рабочую копию, чтобы она соответствовала удаленному репозиторию, используется команда
```Bash
git pull
```
### Git push
Для того чтобы загрузить в удаленный репозиторий свои локальные изменения, используется команда
```Bash
git push
```
