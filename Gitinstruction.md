# Инструкция для работы с Git и удаленными репозиториями

## Что такое Git?
Git - это одна из реализаций распределённых систем контроля версий, имеющая как и локальные, так и удалённые репозитории. Является самой популярной реализацией систем контроля версий в мире.

С помощью Git-a вы можете откатить свой проект до более старой версии, сравнивать, анализировать или сливать свои изменения в репозиторий.

Репозиторием называют хранилище вашего кода и историю его изменений. Git работает локально и все ваши репозитории хранятся в определенных папках на жестком диске.


## 1. Проверка наличия установленного Git
В терминале выполнить команду 

```
git --version
``` 
Если Git установлен, появится сообщение с информацией о версии программы, иначе будет сообщение об ошибке.

## 2. Установка Git
Загружаем последнюю версию программы с [сайта](https://git-scm.com/downloads). 
Устанавливаем с настройками по умолчанию.

## 3. Настройка Git

После установки необходимо «представиться» системе контроля версий. Это нужно сделать всего один раз, и git запомнит вас. Для этого нужно ввести в терминале 2 команды:
```
git config --global user.name «Ваше имя английскими буквами»

git config --global user.email ваша почта@example.com
```
## 4. Инициализация репозитория 
 ```
 git init
 ```
 * Инициализация: указываем папку, в которой git начнет отслеживать изменения
 * В папке создается скрытая папка .git

## 5. Запись изменений в репозитории 
```
 git status
 ```
 * Показывает текущее состояние гита, есть ли изменения, которые нужно закомминить (сохранить)
 ```
 git add <имя файла>
 ```
 * Добавляет содержимое рабочего каталога в индекс (staging area) для последующего коммита. Эта команда дается после добавления файлов. Писать название целиком не обязательно: терминал дозаполнит данные автоматически, для этого необходимо начать вводить название и нажать клавишу `Tab` на клавиатуре

 ```
 git add --all или git add .
 ```
 * Добавить все файлы  проекта в наш будущий коммит

 ```
 git commit -m "комментарий к сохранению"
```
* Зафиксировать или сохранить изменения

## 6. Просмотр истории коммитов
```
git log
```
* Журнал изменений

* Перед переключением версии файла в Git
используйте команду git log, чтобы увидеть
количество сохранений


## 7. Перемещение между коммитами (сохранениями)
```
git checkout
 ```
* Переключение между версиями
* Для работы нужно указать не только
интересующий вас коммит, но и вернуться 
в тот, где работаем, при помощи команды 
`git checkout master`

## 8. Просмотр изменений в текущем файле
 ```
 git diff
 ```
 * Показывает разницу между текущим файлом
и сохранённым

## Оформление текста
**Жирный** шрифт создается при помощи двойного символа `**` или `__` в начале и в конце текста, который необходимо оформить.  
***Жирный курсив*** создается при помощи тройного символа `***` или `___` в начале и в конце текста, который необходимо оформить. 

*Курсивный* шрифт создается при помощи одинарного символа `*` или `_` в начале и в конце текста, который необходимо оформить.  

~~Зачеркнутый текст~~ создается при помощи двойного символа `~~` в начале и конце текста, который необходимо оформить.

## 9. Игнорирование файлов
Для того, чтобы исключить из отслеживания репозитория определенные файлы или папки, необходимо создать файл `.gitignore` и записать в него их названия или шаблоны, соответствующие таким файлам или папкам.

## 10. Создание веток в Git
По умолчанию имя основной ветки в Git - `master`. 
Создать новую ветку можно, воспользовавшись тремя разными командами:
1. Команда 
```brash
git branch <имя новой ветки>
``` 
С её помощью мы создадим ветку, но останемся на той же ветке, на которой были изначально. Чтобы перейти на новую ветку, потребуется ввести дополнительную команду `git checkout <название ветки>`

2. Команда 
```brash
git checkout -b <имя новой ветки>
```
3. Команда 
```brash
git switch -c <имя новой ветки>
```
С помощью команд 2 и 3 мы создадим ветку и сразу же переключимся на нее.

## 11. Посмотреть список веток в репозитории
Для того, чтобы посмотреть список веток в репозитории, необходимо воспользоваться командой
```bash
git log --graph
```
## 12. Переход к другой ветке
Переход от одной ветки на другую осуществляется командой 
```bash
git checkout <название ветки>
```
## 13. Слияние веток
Для того, чтобы слить необходимую ветку с текущей, воспользуемся командой:
```bash
git merge <Имя ветки, которую необходимо слить с текущей>
```
## 14. Разрешение конфликтов 
При слиянии двух веток может возникнуть так называемый **"Конфликт слияния"**. Вызван он тем, что в одном и том же месте, но в разных ветках есть отличная друг от друга информация, а при слиянии веток, информация накладывается друг на друга, тем самым вызывая конфликт.

В самом редакторе конфликт будет подсвечен, а также будут предложены варианты его разрешения:

* Принять текущее изменение 
* Принять входящее изменение
* Принять оба изменения
* Сравнить

Необходимо выбрать один из вариантов и конфликт разрешится.
## 15. Удалить ветку
Для того, чтобы удалить ветку, потребуется использовать команду
```bash
git branch -d <название ветки>
```
## 16. Добавление изображения
Чтобы вставить изображение в текст, достаточно сделать следующее: 
* Скачать изображение и загрузить в рабочую папку редактора
* Создать в редакторе кода новый файл и назвать его `.gitignore`
* Добавить в файл `.gitignore` строку с названием файла, который необходимо игнорировать и его расширение `например: gitlogo.jpeg`
Таким образом мы избегаем того, чтобы в редакторе кода отслеживались изменения с этим файлом.
* Далее воспользуемся кодом:
```bash
[Текст к изображению](относительный путь к изображению)
```

## 17. Инструкция по работе с удаленными репозиториями

### Работа с удаленными репозиториями. Скачивание из текущего репозитория и слияние со своей версией

Освоить работу с удаленными репозиториями, которые находятся не на локальной, а на удаленной машине, например, на сервере.

- **Прежде, чем внести изменения в чужой репозиторий, необходимо создать ответвление репозитория с помощью функциональной кнопки `Fork` на  GitHub**
>Создание копии репозитория позволяет свободно экспериментировать с изменениями без влияния на исходный проект.

- Далее на GitHub необходимо скопировать ссылку на репозиторий, нажав на функциональную кнопку **Code**

**Копировать внешний репозиторий на свой ПК можно командой:** 
```
git clone <скопированная ссылка>
```
- **Работу в скопированном репозитории необходимо продолжить в новой ветке**
> Рекомендуется работать в новой ветке, считая ее черновиком   
>> Работу продолжаем во `встроенном терминале`    
>>> Изменения фиксируем `commit -am <описание изменений>`


- **Для того, чтобы отправить  свою версию в свой GitHub, необходимо воспользоваться командой:**
```
git push
```
>Эта команда позволяет отправить нашу
версию репозитория на внешний
репозиторий. **Требует авторизации 
на внешнем репозитории**

**Команда для предложения изменений, запрос на вливание изменений в репозиторий**
```
pull request
```

## Связывание локального  репозитория с GitHub и работа с ним
* Создать локальный репозиторий
* Создать удаленный репозиторий `create repository`
* Связать локальный репозиторий с удаленным 

*Далее следуем инструкции для работы с созданным локальным репозиторием*

Команды, которые потребуется использовать в редакторе кода:
```
git remote add origin <ссылка на репозиторий>
```
*Добавляем удаленный репозиторий к локальному*

```
git branch -M main
```
*Принудительно переименовываем основную ветку в main*

```
git push -u origin main
```
*Делает ветку активной и отправляет изменения в нее*

Далее возможно перейти на GitHub и внести изменения в репозиторий, после изменения нужно будет удалить или сохранить, выбрав, соответственно `cancel changes` или `commit changes`

Перенести изменения в локальный репозиторий можно с помощью 
```
git pull
```
Внесенные изменения из локального репозитория в удаленный можно с помощью команды 
```
git push
```

>В случае конфликта слияния, разрешение конфликта проходит аналогично тому, как описывалось ранее
