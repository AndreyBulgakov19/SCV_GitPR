# Работа с Git
## 1. Проверка наличия установленного Git
В терминале выполнить команду `Git --version`. Если Git установлен, появится сообщение о версии программы, иначе будет сообщение об ошибке.
## 2. Установка Git
Загружаем последнюю версию Git c [сайта](https://git-scm.com/downloads). Устанавливаем с настройками по умолчанию.
## 3. Настройка Git
При первом использовании Git необходимо представиться. Для этого нужно ввести в терминале две команды:
```Bash
• git config --global user.name «Ваше имя английскими буквами»
• git config --global user.email ваша почта@example.com
```
## 4. Инициализация репозитория
Чтобы начать слежение за существующим проектом, перейдите в папку этого проекта и введите команду `Git init`. В результате существующая папка становится **репозиторием**, и в ней появится скрытая папка с именем **.git**.
## 5. Запись изменений в репозиторий
Чтобы начать управление версиями, необходимо создать новый файл, за которым должна следить система, и выполнить первую фиксацию изменений. Для этого необходимо выполнить команды:
* `git add` <имя файла> - начинает отслеживать добавленный файл и заносит его содержимое в индекс;
* `git commit -m` <"комментарий разработчика"> - фиксирует текущее состояние файла и добавляет комментарий, описывающий закоммиченное изменение.

Если в пректе несколько файлов, которые нужно добавить, то их добавление можно осуществить одной командой `git add .` . 

Если по ошибке добавлен ненужный файл, то изъять его можно при помощи команды `git reset <имя файла>`.
## 6. Проверка состояния файлов
Чтобы узнать текущее состояние Git и есть ли изменения, которые нужно сохранить, необходимо выполнить команду `git status`.
## 7. Просмотр истории версий
Чтобы просмотреть историю всех выполненных фиксаций в проекте, необходимо выполнить команду `git log`.  
В истории содержится вся информация о каждом отдельном коммите, с указанием его хэш-тега, автора, списка изменений и даты, когда они были сделаны.
## 8. Переключение между версиями
Чтобы вернуть выбранный файл к состоянию на момент определенного коммита, необходимо выполнить команду `git checkout <идентификатор коммита>`.  
Чтобы вернуться к актуальному состоянию и продолжить работу, необходимо выполнить команду `git checkout master`.
## 9. Просмотр разницы между версиями
Чтобы вывести разницу между текущим файлом и сохраненным, необходимо выполнить команду `git diff`.
## 10. Игнорирование файлов
Для того, чтобы исключить из отслеживания в репозитории определенные файлы или папки, необходимо создать там файл ***.gitIgnore***  и записать в него их названия или шаблоны, соответствующие таким файлам или папкам. Для проекта достаточно одного файла *.gitIgnore*.  
Чтобы игнорировать все файлы с определенным расширением, то необходимо создать шаблон, используя селектор подстановочного знака __*__, за которым последует расширение файла. Например, нужно игнорировать все файлы, которые имеют расширение *.jpg*, то в файл ***.gitignore***
добавляем следующее:
```Bach
*.jpg
```
Чтобы исключить один файл из игнорирования файлов с одинаковым расширением, нужно воспользоваться шаблоном с отрицанием (восклицательным знаком):
```Bach
*.jpg

!picture.jpg
```
## 11. Создание веток в Git
По умолчанию имя основной ветки в Git - `master`.  
Создать ещё одну ветку можно командой:
```Bash
git branch <имя новой ветки>
```
Данная операция создает ветку, но мы по-прежнему находимся в ветке *master*. Если нам необходимо переместиться на другую ветку, в том числе только что созданную, необходимо использовать команду:
```Bach
git checkout <имя ветки>
``` 
Так же переместиться на другую ветку поможет команда:
```Bach
git switch <имя ветки>
```
Одновременно создать и переключиться на вновь созданную ветку поможет команда:
```Bach
git checkout -b <имя ветки>
```
Чтобы просмотреть список всех имеющихся в проекте веток, используем команду `git branch`. При этом текущая ветка будет отмечена символом __*__.

## 12. Слияние веток и разрешение конфликтов
Для слияния выбранной ветки с текущей нужно выполнить команду:
```Bash
git merge <Название выбранной ветки>
```
Если была изменена одна и та же часть файла в обеих ветках, то может возникнуть конфликт, который потребует участия пользователя.
![Conflict](Change.jpg)
На иллюстрации выше начало места конфликта в файле обозначено так:
```Bach
<<<<<<< HEAD (текущее изменение)
```
Окончание места конфликта в файле обозначено так:
```Bach
>>>>>>> conflict (входящее изменение)
```
Для завершения слияния веток нужно выбрать один из двух вариантов кода. Это можно сделать вручную либо с помощью **подсказок** в редакторе:
1. Принять текущее изменение
2. Принять входящее изменение
3. Принять оба изменения

Также разрешить конфликт слияния можно в редакторе слияния, который приведен на иллюстрации ниже:

![Editor](Redactor.jpg)
**После завершения слияния обязательно нужно закоммитить изменения.**
## 13. Удаление веток
Для удаления ветки из проекта, необходимо выполнить команду:
```Bach
git branch -d <имя ветки>
```
**Нельзя удалить ветку, в которой вы находитесь.** Git выкинет ошибку и не произведет удаление. Следовательно, нужно перейти на другую ветку.  
**Git не позволит удалить ветку, у которой есть несохраненные изменения.** Так мы избегаем ситуации, когда часть написанного кода будет безвозвратно утеряна. Если же мы уверены, что изменения в этой версии не нужны и их можно смело удалять, то вместо флага -d используем -D:
```Bach
git branch -D <имя ветки>
```
## 14. Визуализация всех веток
Отобразить структуру веток в истории коммитов в виде дерева можно с помощью команды:
```Bach
git log --graph
```
## 15. Работа с удаленным репозиторием
### 15.1 Работа со своим репозиторием

**Последовательность действий при переносе локального репозитория в удаленный (на сервис GitHub)**:
1. Создаем аккаунт на сервисе [GitHub](https://github.com/).

2. Создаем локальный репозиторий.

3. Создаем новый репозиторий на GitHub:
* На сайте GitHub кликаем на значок **+** справа сверху или переходим по [ссылке](https://github.com/new).
* Присваиваем новому репозиторию имя **Repository name** и устанавливаем необходимые параметры репозитория.
* Надимаем **Create repository** и переходим на страницу нашего нового репозитория. Сейчас в нем нет никаких файлов (чтобы их выгрузить из локального репозитория, нужно подключиться к удаленному).

4. Добавляем удаленный репозиторий к существующему локальному с помощью команды:
```Bash
git remote add origin <ссылка на удаленный репозиторий>
```
По умолчанию удаленный репозиторий называется **origin**, но можно присваивать любое название.  
**Cсылку** можно взять, нажав на большую зеленую кнопку *Code* на странице репозитория на GitHub.

5. Переименовываем основную ветку локального репозитория в **main** c помощью команды:
```Bach
git branch -M main
```
6. Отправляем наш локальный репозиторий в удаленный (на GitHub), используя команду:
```Bach
git push -u origin name
```
*При первом её использовании нужна авторизация!*

7. Если в удаленный репозиторий внесены изменения с другого компьютера, то скачать их в локальный репозиторий можно командой **git pull**.  Эта команда позволяет скачать все изменения из удаленного репозитория и автоматически сделать merge с нашей версией.

8. Если же изменения были внесены нами в локальный репозиторий, то передать их в удаленный репозиторий можно командой **git push**.

### 15.2 Работа с чужим репозиторием
**Последовательность действий при копировании чужого удаленного репозитория**:

1. Создаем на своем ПК папку, в которую будет клонирован удаленный репозиторий, и открываем её в VS Code.

2. На сервисе GitHub находим нужный нам репозиторий и нажимаем кнопку **Fork**, в результате чего в нашем аккаунте не GitHub создается копия (ответвление) чужого репозитория.

3. Клонируем свою версию удаленного репозитория к себе на компьютер, для этого выполним команду:
```Bach
git clone <ссылка на удаленный репозиторий>
```
**Cсылку** можно получить, нажав на большую зеленую кнопку *Code* на странице скопированного репозитория в нашем аккаунте на GitHub.
4. Перемещаемся в папку со скопированным репозиторием, введя в терминале команду:
```Bach
cd <имя скопированной папки>
```
5. Создаем новую ветку для предлагаемых нами изменений командой **git branch** <имя новой ветки>.

6. Производим все изменения и фиксируем их только в этой новой ветке.

7. Отправляем эти изменения на свой аккаунт командой **git push**.

8. В окне на GitHub делаем запрос на вливание наших изменений в чужой удаленный репозиторий кнопкой **pull request**.

### 15.3 Дополнительные команды для работы с удаленными репозиториями
* Отключение удаленного репозитория от локального:
```Bach
git remote remove <название удаленного репозитория>
```
* Изменение имени удаленного репозитория:
```Bach
git remote rename <старое имя удаленного репозитория> <новое имя удаленного репозитория>
```
* Получение изменений из удаленного репозитория без merge:
```Bach
git fetch [ключи] [имя удаленного репозитория]
```
