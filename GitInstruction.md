## ***1. ПРОВЕРКА НАЛИЧИЯ УСТАНОВЛЕННОГО GIT***

Для того, чтобы проверить установлен ли *GIT* на ПК/проверить версию установленного *GIT*, необходимо ввести в командную строку (открыть __ПУСК__ - вбить __CMD__ - нажать 

![Enter](enter.jpeg))команду:

```
git -v
```
 Инициализация работы всех команд в командной строке осуществляется нажатием, после ввода команды, кнопки __Enter__. Если после инициализации работы команды _git -v_ отобразилась ошибка, значит *GIT* не установлен и вам необходимо перейти в *п. 1.1*

### 1.1 УСТАНОВКА _GIT_

Для того, чтобы установить последнюю версию *GIT*, вам необходимо пройти по [ссылке](https://git-scm.com/downloads) и установить актуальную версию *GIT* в соответствии с операционной системой вашего ПК. После того как файл будет скачан и активируется установка, выберите из предлагаемого списка установку по умолчанию. После завершения установки *GIT* повторите действия из *п. 1*, чтобы убедиться, что *GIT* установлен.

## ***2. НАСТРОЙКА GIT***

Для настройки *GIT* и последующей работе, вам необходимо открыть

 ![Gitbash](gitbash.jpeg), выполнив следующие действия:  __ПУСК__ - вбить __BASH__ - нажать __Enter__. Появившееся черное поле на рабочем столе является командной строкой *GIT*, внутри которой будет осуществляться вся работа с распределенной системой управления версиями. 
Первое, что необходимо сделать, это представиться системе, посредством нижеописанных команд:

```
git config--global user.name 'вбить свое имя'

git config--global user.email 'вбить свой e-mail'

git config--global color.ui auto - данный пункт необязательный, он необходим для включения подсветки
```

Если после того как вы вбили команду и нажали на __Enter__ курсор перешел на новую строку без указания ошибки, значит все сделано правильно. Все данные для регистрации пользователя должны быть вбиты на английском языке.
Для того, чтобы посмотреть данные, которые были внесены, необходимо вбить команду ***git config --list***

## ***3. ИНИЦИАЛИЗАЦИЯ РЕПОЗИТОРИЯ***
Для того что бы создать репозиторий, необходимо перейти в директорию, внутри которой вы будете выполнять свою работу, указав __CD путь к директории__ в командной строке ***GIT BASH***. Создание репозитория в текущей директории осуществляется командой: 

```
git init
```
Вышеуказанная команда создает файл внутри директории *.git*, благодаря чему осуществляется инициализация репозитория внутри системы *GIT*.
Для того, чтобы убедиться, что репозитоий создан, вбиваем команду:
```
git status
```
Данная команда определяет текущее состояние репозитория: на какой ветке мы находимся, наличие коммитов, а также есть ли состояние, которое необходимо закоммитить.

## ***4. ЗАПИСЬ ИЗМЕНЕНИЙ В РЕПОЗИТОРИЙ***

После того, как вы закончили этап работы внутри репозитория, вам необходимо зафиксировать изменения из текущего состояния индекса. Для начала, воспользуемся командой *git status*, чтобы ознакомиться с файлами, готовыми для создания коммита. Вам необходимо добавить файлы, с которыми вы работали в индекс командой:
```
git add имя файла
```
Следующий шаг - запись изменений в репозиторий.
Если файлы только созданы и еще не отслеживаются, то вам необходимо воспользоваться командой:
```
git commit -m 'подпись коммита'
```
Вы можете также воспользоваться командой 
```
git commit
```
Выполнив данную команду, у вас откроется редактор, настроенный по умолчанию, для подписи коммита. После завершения работы, необходимо нажать клавишу *Esc* и вбить __:wq__.

Вы можете также воспользоваться командой:
```
git commit -a
```
Данная команда создает коммит, с одновременным добавлением в него измененных файлов и открывает редактор для подписи коммита.

Если вы работали с с уже отслеживаемыми файлами (ранее добавленными в индекс, но измененными), то можно воспользоваться командой
```
git commit -am 'подпись коммита'
```
Вышеуказанная команда создает коммит с одновременным добавлением в него измененных файлов без вызова редактора, с указанной в апострофах (можно использовать и кавычки) подписью.

После того как коммит сделан, рекомендуется запустить *git status*, чтобы убедиться, что все изменения зафиксированы.

## ***5. ПРОСМОТР ИСТОРИИ КОММИТОВ***

Для того, чтобы получить доступ к истории репозитория, вам необходимо в *Git Bash* вбить команду:
```
git log
```
После выполнения данной команды вы получите всю информацию о каждом коммите (уникальный номер, автор, время создания, записанный комментарий), созданном в текущем репозитории. 

Если у вас много коммитов и вы хотите получить историю коммитов в более компактном варианте, то воспользуйтесь командой:
```
git log --oneline
```

История создания коммитов отображается в хронологическом порядке сверху - последний, снизу - первый. **HEAD** - текущее состояние, в котором вы находитесь.

Если вы хотите познакомиться со всеми операциями, которые были совершены во время работы в репозитории, вам необхожимо воспользоваться командой:
```
git reflog
```
Посредством этой команды вы увидите, все операции, которые были совершены в текущем репозитории, в том числе, были ли совершены отмены коммита, переход по истории и т. д.

Если вы хотите посмотреть какие изменения были внесены относительно последнего зафиксированного состояния, воспользуйтесь командой:
```
git diff
```

## ***6. ПЕРЕМЕЩЕНИЕ МЕЖДУ СОХРАНЕНИЯМИ (КОММИТАМИ)***

Для того, чтобы вернуться на состояние предыдущего коммита, необходимо воспользоваться командой:
```
git reset [уникальный номер коммита, на который мы хотим вернуться]
```
Данная команда выполняет сброс коммита, если вы хотите отменить именно последствия коммита, то необходимо отменить действия, которые были сделаны в коммите посредством команды:
```
git revert [уникальный номер коммита, действие которого необходимо отменить]
```
Данная операция создает новый коммит, который отменяет действия, совершенные предыдущим, запуская редактор коммита, в котором фиксируется _revert_ коммита. Для того, чтобы выйти из редактора, необходимо нажать клавишу ![Esc](esc.png) и вбить __:wq__.

Воспользуйтесь *п. 5*, для того, чтобы просмотреть историю операций с коммитами, а также *git status*, чтобы проверить текущее состояние репозитория.

## ***7. ИГНОРИРОВАНИЕ ФАЙЛОВ***

Для того, чтобы скрыть файлы из репозитория, необходимо создать файл в текущем репозитории:
```
.gitignore
```
Данный файл используется для того чтобы игнорировать файлы от попадания в индекс репозитория.
Внутри созданного файла необходимо указать правила игнорирования, используя один из указанных ниже вариантов: 

* указать полное наименование файлов, которые необходимо игнорировать

* указать расширение файлов, которые необходимо игнорировать

Для того, чтобы внести изменения внутри файла (любого, не только в *.gitignore*), необходимо в командной строке *GIT BASH* вбить команду:
```
echo 'указываем то, что хотим внести внутрь файла'>[название файла]
```

Для того, чтобы убедиться, что изменения в файл внесены, воспользуемся командой:

```
cat [указываем название файла, содержимое которого хотим посмотреть]
```

После чего записываем изменения в репозитории согласно п. 4 данной инструкции.

## ***8. СОЗДАНИЕ ВЕТОК В GIT***

Для того, чтобы определить какие ветки нам доступны на текущий момент, необходимо использовать команду:
```
git branch
```
Звездочка (*) перед названием ветки говорит нам на какой ветке мы находимся в данный момент. 

Перед тем как начать работать, обязательно контролируйте, на какой ветке вы вносите изменения. В *GIT BASH* ветка указывается в скобках () в конце пути к директории.

Для того, чтобы создать новую ветку, необходимо воспользоваться командой:
```
git branch [указать название ветки]
```
Если вам необходимо создать ветку, находясь при этом не на той ветке, от которой необходимо создать новую, то можно воспользоваться следующей командой:
```
git branch [указать название новой ветки] [указать название ветки, от которой будет создана новая]
```

Для того, чтобы переключится на нужную вам ветку, воспользуйтесь командой:
```
git checkout [указать название ветки]
```
Вы можете воспользоваться командой, которая осуществляет создание и переключение ветки:
```
git checkout -b [указать название ветки]
```
После выполнения последней команды вы будете находиться внутри созданной ветки.

## ***9. СЛИЯНИЕ ВЕТОК И РАЗРЕШЕНИЕ КОНФЛИКТОВ***

### 9.1 СЛИЯНИЕ ВЕТОК

* Для того, чтобы осуществить слияние веток, необходимо переключиться на ветку, в которую будут влиты изменения из другой ветки, используя рекомендации п. 8 данной инструкции.

После того, как вы убедились, что вы находитесь на нужной вам ветке, необходимо выполнить слияние веток командой:
```
git merge [указать название ветки, коммиты которой будете вливать в текущую]
```

### 9.2 РАЗРЕШЕНИЕ КОНФЛИКТОВ

* При работе на разных ветках могут появиться ситуации, при которых 1 строка внутри файла будет содержать разную информацию относительно веток репозитория. В таком случае, при использовании команды *git merge*, произойдет конфликт слияния, так как _GIT_ не сможет автоматически определить, какое изменение является правильным. 

Вам необходимо __вручную__ разрешить конфликт:

* Строка __=======__ является «центром» конфликта. 

Все содержимое между этим центром и строкой ***<<<<<<< HEAD*** находится в текущей ветке, на которую ссылается указатель *HEAD*. 

А все содержимое между центром и строкой ***>>>>>>> [название ветки]_to_merge_later*** является содержимым ветки для слияния.

* Вам необходимо оставить только ту часть, которая является правильной, удалив вручную неактуальную часть.

После того как конфликты разрешены и слияние веток выполнено, необходимо зафиксировать текущее состояние репозитория в соответствии с п. 4 данной инструкции.

Результат слияния веток наглядно демонстрируется через команду:
```
git log --graph
```

Если вы поняли, что не можете разрешить текущие конфликты, то слияние необходимо отменить командой:
```
git merge --abort
```

## ***10. УДАЛЕНИЕ ВЕТОК***
Для того, чтобы удалить ветку, необходимо воспользоваться командой:
```
git branch -d [указать название ветки, которую необходимо удалить]
```
Если при работе с веткой вы не смогли завершить слияние, либо были другие ошибки при работе, то вышеуказанная команда не сработает. В этом случае, если вы уверены, что хотите все же удалить ветку, то необходимо использовать команду:
```
git branch -D [указать название ветки, которую необходимо удалить]
```

Опция _-D_ обозначает ***--delete --force***, которая удаляет ветку независимо от ее статуса *push* или *merge*, так что будьте осторожны при её использовании.

## ***11. РАБОТА С УДАЛЕННЫМИ РЕПОЗИТОРИЯМИ, PULL REQUEST***

### 11.1 РАБОТА С УДАЛЕННЫМИ РЕПОЗИТОРИЯМИ

Для того, чтобы получить доступ к работе с удаленными репозиториями, необходимо создать свой аккаунт на [сайте](https://github.com).

Пройдя регистрацию аккаунта на _GitHub_, вам необходимо создать репозиторий, следуя инструкциям данного ресурса. После того, как вы создали репозиторий, убедитесь, что вы находитесь внутри него, а также изучите возможности работы с репозиторием на _GitHub_.

Для работы с удаленным репозиторием вам необходимо нажать на кнопку code и скопировать путь к репозиторию на вкладке _HTTPS_:


Для того, чтобы работать локально с репозиторием, созданным на _GitHub_, выполните команду в _GIT BASH_:
```
git clone [вставьте скопированный путь к репозиторию на вкладке HTTPS в GitHub] [указать папку, куда будет клонирован репозиторий, либо выполняйте клонирование, находясь в нужной вам папке]
```

Выполнив вышеуказаннную команду, вы создадите клон репозитория, созданного на _GitHub_. 

Для того, чтобы узнать какие удаленные репозитории настроены для работы локально, выполните команду:

```
git remote
```

Данная команда предоставит вам список удаленных репозиториев.

Для работы с клонированной версией репозитория рекомендуется использовать отдельные ветки, создание которых описано в п. 8 данной инструкции.

После того, как вы завершили работу на локальной версии удаленного репозитория, необходимо загрузить изменения в репозиторий _GitHub_ с помощью команды:
```
git push
```
При выполнении данной команды появится сообщение об ошибке, так как в версии репозитория на _GitHub_ отсутствует ветка, которую вы создали. Для того, чтобы исправить ошибку, существует два способа:

* зайти в репозиторий на _GitHub_ и создать ветку, наименование которой соответствует вашей локальной версии
* выполнить одну из предложенных команд:
```
git push --set-upstream origin [название ветки, которую вы создали] 

git push -u origin [название ветки, которую вы создали]
```
Выполнив одну из предложенных команд произойдет автоматическое создание ветки в _remote-репозитории_, наименование и содержание которой будет соответствовать локальной версии репозитория.
***origin*** - это _URL адрес remote-репозитория_.
Для того, чтобы увидеть измененный репозиторий на _GitHub_, обновите страницу. Последние изменения будут выделены цветом.

Если вам необходимо загрузить изменения из удаленного репозитория, то воспользуйтесь командой:
```
git pull
```

Воспользуйтесь командой _git log_, для того, чтобы ознакомиться со внесенными изменениями локальной версии из _remote-репозитория_.

Для того, чтобы обновить текущую синхронизацию по данным об удаленном репозитории, воспользуйтесь командой:
```
git fetch
```
