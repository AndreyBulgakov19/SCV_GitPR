# Работа с Git

## 1. Проверка наличия установленного Git

В терминале выполнить команду 
```
git --version
```
Если Git установлен появится сообщение с информацией о версии программы, иначе будет сообщение об ошибке.

## 2. Установка Git

Загружаем последнюю версию Git с [сайта](https://git-scm.com/downloads).
Устанавливаем с настройками по умолчанию. 

## 3. Настройка Git

При первом использовании Git необходимо представиться. Для
этого нужно ввести в терминале 2 команды:
```
git config --global user.name "Ваше имя английскими буквами" 
git config --global user.email "Ваша почта@example.com"
```

## 4. Инициализация репозитория

Перед началом работы с репозиторием его необходимо инициализировать командой
```
git init
```
Теперь Git отслеживает изменения проекта.

Успешную инициализацию репозитория можно проверить командой

```
git status
```

При выполнении команды должно вывестись сообщение типа `"On branch master"`.

## 5. Запись изменений в репозиторий

Псоле инициализации репозитория для создания commit необходимо добавить файлы. Для добавления всех файлов текущего репозитория необходимо выполнить команду

```
git add .
```

или
```
git add --all
```
Для добавления конкретного файла необходимо выполнить команду

```
git add <имя файла>
```

После добавления файла можно приступать к записи commit. Делается это с помощью команд

```
git add <имя файла>
git commit -m "Комментарий к commit"
```

или 

```
git commit -a -m "Комментарий к commit"
```

Изменить комментарий поседнего записанного commit можно командой

```
git commit -ament -m "Комментарий к commit"
```

Для проверки всех изменений, которые вносились в файл после последней фиксации, существует команда

```
git diff
```

## 6. Просмотр истории commit.

Для просмотра всех commit, записываемых в процессе создания проекта с помощью команд, указанных в разделе 5, можно воспользоваться командой

```
git log
```
После выполнения команды будет выведено сообщение типа

```
commit b18fcfcf583d63af12f3df732be59a2f187e0e03
Author: Andrey Kozikov <nwisdom@mail.ru>
Date:   Sat Sep 30 16:08:30 2023 +0500

    Добавили раздел 4 - Инициализация репозитория

commit f719a4baa20fcea5310843ada87e7926b74d0793
Author: Andrey Kozikov <nwisdom@mail.ru>
Date:   Sat Sep 30 15:48:57 2023 +0500


    Добавили раздел 3 - Настройка  Git
```
Для вывода только комментариев к commit можно воспользоваться командой

```
git log --oneline
```
Для графического представления используется команда 
```
git log --graph
```
Команда также может использрваться с параметром "**--goneline**" для сокращенного вывода информации.

## 7. Перемещение между сохранениями

Для переключения к одному из ранее сохраненных commit в терминале надо ввести команду (обозначение commit можно получить одной из команд, указанных в разделе 6)

```
git checkout <обозначение commit>
```

Для возвращения к актуальной версии необходимо в терминале ввести команду
```
git checkout master
```
или 
```
git switch master
```

## 8. Игнорирование файлов
Для того, чтобы исключить из отслеживания в репозитории файла или папки необходимо создать файл ***.gitignore*** и добавить в него их названия или шаблоны, соответствующие таким файлам или папкам.

## 9. Создание веток в Git
По умолчанию имя основной ветки в Git - **master**.
Создать ветку можно командой
```
git branch <имя новой ветки>
```
Список веток в репозитории можно посмотреть с помощью команды `git branch`.
Текущая ветка будет отмечена звездочкой: **\* master**

Список веток в репозитории можно посмотреть с помощью команды `git branch`. 
Текущая ветка будет отмечена звездочкой: **\*  master.**

Создать новые ветки можно также командами
```
git checkout -b <имя ветки>
```
или
```
git branch -c <имя ветки>
```
Выполнение обеих команд приведет к созданию новой ветки и переключению на нее.

## 10. Слияние веток в Git и разрешение конфликтов
Для слияния веток в Git используется команда `git merge <имя ветки>`
Перед выполнением команды необходимо переключиться в ветку **master**.

Если в ветке **master** и в ветке, которую мы хотим слить содержится одинаковый текст, отличающийся друг от друга некоторыми элементами, то возникнет конфликт. Перед фиксацией изменений этот конфликт необходимо разрешить путем выбора тех изменений, которые необходимо оставить, все остальные элементы будут удалены.

## 11. Удаление веток
Для удаления веток в Git используется команда 
```
git branch -d <имя ветки>
```
Удалить ветку можно только находясь в другой ветке (удаление ветки из самой себя завершится ошибкой и ветка удалена не будет).

Перед удалением ветки она должна быть слита с веткой **master**.

Для принудительного удаления ветки без слияния используется команда
```
git branch -D <имя ветки>
```

## 12. Работа с удаленными репозиториями.
Для работы с удаленным репозитерием, например расположеным на сайте GitHub, необходимо выполнить следующие действия:

1. Создать локальный репозиторий
2. Связать удаленный репозиторий с локальным.

Подключение удаленного репозитория к локальному осуществляется командой
```
git remote add <имя удаленного репозитория> <ссылка на удаленный репозиторий>
```
Дальнейшая работа с локальным репозиторием будет проходить под именем `<имя удаленного репозитория>`.

Перед началом работы необходимо выполнить клонирование удаленного репозитория к себе на копьютер командой
```
git clone <ссылка на удаленный репозиторий>
```
Получение изменений с удаленного репозитория производится командой 
```
git pull <имя удаленного репозитория>
```
Команда получает изменения из переданного удаленного репозитория и обновляет рабочую копию в соответствии с удаленным репозиторием. По умолчанию слияние удаленной ветки с локальной происходит именно в fast-forward режиме.

Отправка изменений в удаленный репозиторий осуществляется командой
```
git push <имя удаленного репозитория>
```
Если на удаленной ветке есть коммиты, которые были сделаны позже наших, то есть слияние в режиме ``fast-forward`` выполнить невозможно, то команда завершится с ошибкой. В таком случае рекомендуется получить изменения с удаленного репозитория и провести объединение с локальным, после чего повторить отправку изменений в удаленный репозиторий.
