# Работа с Git
![логотип](gitlogo.png)
## **Первоначальная настройка и работа**
### 1. Установка Git
Загрузить последнюю версию программы с сайта https://git-scm.com/downloads с настройками по умолчанию
### 2. Установка VSCode 
Загрузить последнюю версию программы с сайта https://code.visualstudio.com/Download
### 3. Открытие терминала в VSCode
Вид -> терминал
### 4. Переход на следующую строку в терминале
В терминале нажать "q"
### 5. Проверка наличия установленного Git
В терминале выполнить команду `git --version`. Если Git установлен, появится сообщение с информацией о версии программы, иначе - сообщение об ошибке
### 6. Проверка наличия и состояния репозитория
В терминале выполнить команду `git status`, если репозитория нет, будет выведена информация об этом, иначе - информация о ветке, изменениях
### 7. Создание репозитория
В терминале выполнить команду `git init`, будет указано его месторасположение
### 8. Создание нового файла
В проводнике слева нажать на иконку создать файл напротив названия папки, дать ему имя с расширением md
### 9. Добавить новый файл в отслеживание
В терминале выполнить команду `git add <название файла>`
### 10. Предпросмотр оформления 
Напротив названия файла справа сверху нажать на два прямоугольника с лупой
### 11. Сохранение изменений
В терминале выполнить команды:

`git commit -am "сообщение"` если изменения в одном файле

`git commit -m "сообщение"` если измменения не в одном файле
### 12. Узнать, какие произошли несохраненные изменения
В терминале выполнить команду `git diff`.
### 13. Узнать информацию о сохраненных изменениях 
В терминале выполнить команду:

 `git log` - будут выведены изменения с версией изменения, датой, именем исполнителя, и сообщением от него. 

`git log --oneline` - будут выведены сообщения о версиях в столбец
### 14. Переход к одной из сохраненных версий
В терминале выполнить команду `git checkout <номер_версии>`, и произойдет автоматическое сохранение этого файла на компьютере в этой версии
### 15. Возвращение к новейшей версии
 В терминале выполнить команду `git switch -`
## **Работа с ветками**
### 16. Создание ветки из промежуточного сохранения
 В терминале выполнить команду `git checkout -c <название_ветки>`
### 17. Справочник базовых команд
 В терминале выполнить команду `git help`
### 18. Получить помощь по деталям определенной команды
В терминале выполнить команду `<команда> -h`
### 19. Игнорирование файлов
Для того чтобы исключить из отслеживания в репозитории определенные файлы или папки, необходимо создать файл `.gitignore` и записать в него их названия или шаблоны
### 20. Создание веток
По умолчанию имя основной ветки - master. 
Создать новую ветку:
`git branch <имя новой ветки>`

Список веток можно посмотреть с помощью команды `git branch`
### 21. Слияние веток
Когда работа в дополнительной ветке завершена, нужно сохранить изменения,  перейти в ветку, к которой нужно присоединить дополнительную и использовать в терминале команду `git merge <имя ветки>`
### 22. Разрешение конфликтов
 Если произойдет конфликт(была введена информация в сливающейся ветке в том же месте, где была введена другая информация в другой ветке), git выведет информацию о том, что произошел конфликт. В такой ситуации нужно самостоятельно определить, что будет получено. Варианты, предлагаемые git: оставить изменения основной ветки, оставить изменения дополнительной ветки, оставить все изменения, сравнить их.
### 22. Удаление слитой ветки
После ввода команды будет выведен ранее принадлежащий ей номер.
Чтобы удалить слитую ветку, нужно ввести в терминале команду `git branch -d <имя ветки>`
### 23. Удаление неслитой ветки
При попытке удалить неслитую ветку, git выведет предупреждение об этом и предложит ее удаление без слияния с утерей изменений (для этого нужно ввести в терминале команду `git branch -D <имя ветки>`) и даст возможность отключить это уведомление. 
## **Работа с удаленными репозиториями**
### 24. Создание аакаунта на GitHub.com
### 25. Использование удаленного репозитория
Необходимо найти на GitHub.com необходимый репозиторий через имя автора и название работы, скопировать ссылку на него и в терминале выполнить команду `git clone <ссылка>`
### 26. Публикация локального репозитория
1.  Создать репозиторий, привязать папку и создать один файл, провести в нем как минимум один коммит
2. При первом связывании необходимо авторизоваться
3. В правом верхнем углу GitHub.com нажать "+", "Новый репозиторий", дать ему имя, выбрать настройки
GitHub предложит несколько вариантов: создать новый репозиторий через терминал, привязать существующий локальный репозиторий и импортировать код из GitHub
4. Выбрать второй вариант, использовать предложенный код в терминале:

git remote add origin <ссылка>

git branch -M main

git push -u origin main A
Репозиторий будет выложен в текущем виде и последующие изменения не будут в нем отображаться
### 27. Вывод изменений в удаленный репозиторий на GitHub
В терминале выполнить команду `git push`
### 28. Работа с другими людьми
Изменять репозиторий можно в нем в GitHub
Чтобы изменения, проведенные через него другим человеком или самостоятельно, отобразились в локальном репозитории, нужно выполнить в терминале команду `git pull`
### 29. Направление pull request
1. Полностью скопировать себе нужный репозиторий из GitHub с помощью кнопки "fork" справа с краю от имени выложившего
2. Клонировать репозиторий через ссылку на уже свой репозиторий, создав два репозитория 
3. Для внесения изменений создать новую ветку
4. На новой ветке создать файл "README.md", работать в нем
5. Отправить изменения на свой удаленный репозиторий
6. На GitHub появится новая ветка в репозитории и отправить запрос на слияние
### 30. Перемещение между репозиториями и папками
В терминале выполнить команду `cd <имя папки>`
