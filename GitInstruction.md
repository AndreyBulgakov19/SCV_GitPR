# **ИНСТРУКЦИЯ ПО РАБОТЕ С _GIT_**
---
## ***1. ПРОВЕРКА НАЛИЧИЯ УСТАНОВЛЕННОГО GIT***

Для того, чтобы проверить установлен ли GIT на ПК/проверить версию установленного GIT, необходимо ввести в командную строку (открыть __ПУСК__ - вбить __CMD__ - нажать __Enter__) команду:

```
git -v
```
 Инициализация работы всех команд в командной строке осуществляется нажатием, после ввода команды, кнопки __Enter__. Если после инициализации работы команды _git -v_ отобразилась ошибка, значит *GIT* не установлен и вам необходимо перейти в *п. 1.1*

### 1.1 УСТАНОВКА _GIT_

Для того, чтобы установить последнюю версию *GIT*, вам необходимо пройти по [ссылке](https://git-scm.com/downloads) и установить актуальную версию *GIT* в соответствии с операционной системой вашего ПК. После того как файл будет скачан и активируется установка, выберите из предлагаемого списка установку по умолчанию. После завершения установки *GIT* повторите действия из *п. 1*, чтобы убедиться, что *GIT* установлен.

## ***2. НАСТРОЙКА GIT***

Для настройки *GIT* и последующей работе, вам необходимо открыть *GIT BASH*, выполнив следующие действия:  __ПУСК__ - вбить __BASH__ - нажать __Enter__. Появившееся черное поле на рабочем столе является командной строкой *GIT*, внутри которой будет осуществляться вся работа с распределенной системой управления версиями. 
Первое, что необходимо сделать, это представиться системе, посредством нижеописанных команд:

```
git config--global user.name 'вбить свое имя'

git config--global user.email 'вбить свой e-mail'

git config--global color.ui auto - данный пункт необязательный, он необходим для включения подсветки
```

Если после того как вы вбили команду и нажали на __Enter__ курсор перешел на новую строку без указания ошибки, значит все сделано правильно. Все данные для регистрации пользователя должны быть вбиты на английском языке.
Для того, чтобы посмотреть данные, которые были внесены, необходимо вбить команду ***git config --list***

## ***3. ИНИЦИАЛИЗАЦИЯ РЕПОЗИТОРИЯ***
Для того что бы создать репозиторий, необходимо перейти в директорию, внутри которой вы будете выполнять свою работу, указав __CD путь к директории__ в командной строке ***GIT BASH***. Создание репозитория в текущей директории осуществляется командой: 

```
git init
```
Вышеуказанная команда создает файл внутри директории *.git*, благодаря чему осуществляется инициализация репозитория внутри системы *GIT*.
Для того, чтобы убедиться, что репозитоий создан, вбиваем команду:
```
git status
```
Данная команда определяет текущее состояние репозитория: на какой ветке мы находимся, наличие коммитов, а также есть ли состояние, которое необходимо закоммитить.

## ***4. ЗАПИСЬ ИЗМЕНЕНИЙ В РЕПОЗИТОРИЙ***

После того, как вы закончили этап работы внутри репозитория, вам необходимо зафиксировать изменения из текущего состояния индекса. Для начала, воспользуемся командой *git status*, чтобы ознакомиться с файлами, готовыми для создания коммита. Вам необходимо добавить файлы, с которыми вы работали в индекс командой:
```
git add имя файла
```
Следующий шаг - запись изменений в репозиторий.
Если файлы только созданы и еще не отслеживаются, то вам необходимо воспользоваться командой:
```
git commit -m 'подпись коммита'
```
Вы можете также воспользоваться командой 
```
git commit
```
Выполнив данную команду, у вас откроется редактор, настроенный по умолчанию, для подписи коммита. После завершения работы, необходимо нажать клавишу *Esc* и вбить __:wq__.

Вы можете также воспользоваться командой:
```
git commit -a
```
Данная команда создает коммит, с одновременным добавлением в него измененных файлов и открывает редактор для подписи коммита.

Если вы работали с с уже отслеживаемыми файлами (ранее добавленными в индекс, но измененными), то можно воспользоваться командой
```
git commit -am 'подпись коммита'
```
Вышеуказанная команда создает коммит с одновременным добавлением в него измененных файлов без вызова редактора, с указанной в апострофах (можно использовать и кавычки) подписью.

После того как коммит сделан, рекомендуется запустить *git status*, чтобы убедиться, что все изменения зафиксированы.

## ***5. ПРОСМОТР ИСТОРИИ КОММИТОВ***

Для того, чтобы получить доступ к истории репозитория, вам необходимо в *Git Bash* вбить команду:
```
git log
```
После выполнения данной команды вы получите всю информацию о каждом коммите (уникальный номер, автор, время создания, записанный комментарий), созданном в текущем репозитории. 

Если у вас много коммитов и вы хотите получить историю коммитов в более компактном варианте, то воспользуйтесь командой:
```
git log --oneline
```

История создания коммитов отображается в хронологическом порядке сверху - последний, снизу - первый. **HEAD** - текущее состояние, в котором вы находитесь.

Если вы хотите познакомиться со всеми операциями, которые были совершены во время работы в репозитории, вам необхожимо воспользоваться командой:
```
git reflog
```
Посредством этой команды вы увидите, все операции, которые были совершены в текущем репозитории, в том числе, были ли совершены отмены коммита, переход по истории и т. д.

Если вы хотите посмотреть какие изменения были внесены относительно последнего зафиксированного состояния, воспользуйтесь командой:
```
git diff
```

## ***6. ПЕРЕМЕЩЕНИЕ МЕЖДУ СОХРАНЕНИЯМИ (КОММИТАМИ)***

Для того, чтобы вернуться на состояние предыдущего коммита, необходимо воспользоваться командой:
```
git reset [уникальный номер коммита, на который мы хотим вернуться]
```
Данная команда выполняет сброс коммита, если вы хотите отменить именно последствия коммита, то необходимо отменить действия, которые были сделаны в коммите посредством команды:
```
git revert [уникальный номер коммита, действие которого необходимо отменить]
```
Данная операция создает новый коммит, который отменяет действия, совершенные предыдущим, запуская редактор коммита, в котором фиксируется _revert_ коммита. Для того, чтобы выйти из редактора, необходимо нажать клавишу *Esc* и вбить __:wq__.

Воспользуйтесь *п. 5*, для того, чтобы просмотреть историю операций с коммитами, а также *git status*, чтобы проверить текущее состояние репозитория.
