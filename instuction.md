# Работа с Git 
## 1. Проверка наличия установленного GIT
В терминале выполнить команду `git --version`

текст в апострофе- подсветка

```ntrcn```

Если Git установлен, появится инфа о версии программы.
## 2. Установка Git
Загружаем с [сайта](https://git-scm.com/downloads) последнюю версию Git.
##  Настройка Git
При первом использовании Git необходимо представиться.
Для этого нужно ввести в терминале 2 команды:
```
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```
git reflog команда показывает все перемещения, вернутся switch -

## 4. Инициализация репозитория
использовать команду *git init*
после того, как папка связалась с репозиторием создаем новый файл, именуем его, присваиваем расширение.


## 5. Запись изменений в репозиторий
используем команду *git add <имя файла>*. Это подготовка к отслеживанию. используем TAB для автозаполнения.
Фиксируем изменения в репозитории с помощью команды *git commit -m 'и здесь пишем что было добавлено'*.

## 6. Просмотр истории коммитов
чтобы посмотреть что же было зафиксировано использовать команду *git diff или git log* (`чтобы выйти нажать Q`)

## 7. Перемещение между сохранениями
вывести список всех сохранений командой **git log --one line**
затем находим *хэш код* версии, куда надо перейти
используем команду **git checkout и копируем хэшкод**. Так мы `переходим` на другую ветку.
`Возвращаемся` в ветку master командой *git switch -*
Либо можно создать новую ветку командой *git switch -c <название новой ветки>*

Чтобы `добавить изображение`, надо положить картинку в папку на рабочем столе, затем в VSCode пишем команду ![любое название](имя картинки)

## 8. Игнорирование файлов
Создать **.gitignore** где будут собраны игнорируемые файлы, чтобы исключить из отслеживания определенные файлы или папки. Можно указать расширение, и все файлы с таким расширением автоматически попадут в git ignore. Использовать команду *.png (любое другое расширение).

## 9. Создание веток

Чтобы добавить новую ветку используют команду **git branch имя ветки**.
Или `вариант №2 создания новой ветки и сразу же перейти в нее`: команда **git checkout -b название ветки**.

Чтобы `переключиться на другую ветку` из текущей ветки использовать команду **git checkout имя ветки**.
Чтобы посмотреть `список всех веток` использовать команду **git branch**.
Чтобы вернутся в ветку master использовать команду ``git switch master``.
``````
Текущая ветка будет отмечена звездочкой: *master
``````
## 10. Слияние веток и разрешение конфликтов
Для слияния выбранной ветки с текущей использовать команду **git merge название сливаемой ветки**

После разрешения конфликта нужно выполнить коммит слияния:
``````
git commit -am ' ... '
``````

Если была изменена одна и та же часть файла в обеих ветках, то может возникнуть `конфликт`, который потребует участия пользователя. VSCode предлагает варианты разрешения конфликта.

## 11. Удаление веток
Дополнительные ветки- это черновики. После слияния с веткой master дополнительные ветки нужно удалять.

**Нельзя удалить ветку, находясь в ней же.**

Для того, чтобы удалить ветку, сначала надо перейти в ветку *master. Для этого используем команду **git switch master**.
Затем используем команду удаления: **git branch -d имя ветки, которую удаляем.**

Если ветки *master и удаляемая ветка не слиты, git выдаст ошибку.

## 12. Работа с удаленными репозиториями
1. Создать аккаунт на Github
2. Создать локальный репозиторий
3. Создать удаленный репозиторий
4. Связать удаленный репозиторий с локальным
5. Добавить удаленный репозиторий к проекту на локальном компе:
``````
git remote add  <имя репозитория> <url адрес репозитория в сети>
``````

Для получения и слияния изменений из удаленного репозитория:

**git pull**

Посмотреть список удаленных репозиториев:

**git remote**
